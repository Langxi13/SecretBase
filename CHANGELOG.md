# 变更记录

## 5.3.1 - 2026-07-18

### 跨设备配对与交互补齐

- Android 加密快照同步新增二维码扫描器；扫码后只填充 WebDAV 地址、用户名和 V2 恢复码，WebDAV 应用密码仍必须由用户手动输入。
- Android 加入页新增配对链接粘贴、独立连接测试、恢复码安全复制和配对链接复制；成功加入后立即清除表单中的应用密码与恢复码。
- Web 和桌面端加入页支持直接粘贴 `secretbase://sync/join` 配对链接，并自动识别 V1/V2 协议、WebDAV 地址、用户名和恢复码。
- 新生成的配对链接不再重复携带裸同步密钥，改为携带已经包含空间身份、同步密钥和校验和的恢复码；仍不包含 WebDAV 应用密码。
- Android 保持对 V5.3.0 旧版 V2 二维码的兼容，可在本机将旧链接中的空间身份与密钥转换为相同恢复码。
- 锁定 Vault 时 Android 同步弹窗会清除应用密码、恢复码、配对材料和冲突上下文，避免解锁后恢复旧敏感界面。
- Android 发布门禁新增摄像头权限和扫码依赖检查，并补充配对 URI、旧二维码兼容、HTTP/嵌入凭据拒绝及前端导入回归测试。
- Web/桌面端加入页补充恢复码校验和旧版 `key=` 配对链接转换；Android 同步弹窗拆分为独立 UI 模块，并在锁定、断开和粘贴后清理敏感状态。

## 5.3.0 - 2026-07-18

### 坚果云兼容的加密快照同步

- 新增 Sync Protocol V2：使用 `SBS2` 端到端加密全量快照和不可变 DAG，不依赖强 ETag、条件写入或可变 head，兼容只提供基础 WebDAV 能力的坚果云等服务。
- Web、Windows、macOS 和 Android 均支持创建/加入同步空间、自动同步、历史查看、恢复、并发分支合并、脱敏冲突处理和密钥轮换。
- 同步设置支持修改连接、设备名和自动同步开关；解锁及 Vault revision 变化后使用 5 秒防抖自动同步，锁定时会取消活动 WebDAV 请求，手动与自动同步共用全局互斥。
- 三台及以上设备同时修改时改为分阶段处理冲突，未处理分支不会被静默标记为已合并。
- 新增显式“压缩并清理历史”：只有本机与远端基线完全一致时才允许执行；操作创建新的根快照，其他设备须使用新恢复码重新加入，旧空间清理失败不会影响新空间。
- Android 新增 WebDAV 命名空间解析、有限重试、无 ETag 上传校验和同步历史压缩；恢复码、历史恢复、密钥轮换、压缩和远端删除均要求主密码，高风险操作还要求一次性确认令牌与确认词。

### 验证与兼容

- 增加 Python、Rust、Flutter WebDAV/XML、三分支冲突、压缩清理和弱网回归测试，并将 V2 端到端检查纳入发布门禁。
- 修复历史压缩在快照内容相同但顶层条目顺序不同时误报基线不一致的问题；顶层条目按 ID 集合比较，条目内部字段顺序仍严格保留。
- 单个快照的密文和解压文档继续限制为 64 MiB；远端发现额外限制为最多 1000 个快照、累计 256 MiB 密文和 256 MiB 解密内容，并限制并发 frontier 为 32。
- 远端压缩或删除会重新枚举并在发现未知对象、并发新增或删除失败时停止；弱 WebDAV 无法提供删除事务，本机 Vault 始终保留，失败不会被误报为远端已清空。
- 修复 Android 加入同步空间后首次合并快照上传失败时仍停留在未完成加入状态的问题；本机加入会保留，并在下次同步继续发布合并结果。
- 将 V2 同步进一步拆分为共享协议、冲突处理、管理操作、核心服务和远端仓库模块，并为各模块增加体量门禁。
- V1 严格 ETag 同步路径保持不变；V1 用户可通过显式迁移创建独立 V2 空间，旧空间不会自动删除。
- 远端仍只看到加密对象、UUID、大小和访问时间；字段名、字段值、主密码、WebDAV 密码和 AI 数据不会上传到同步服务。

## 5.2.0 - 2026-07-17

### WebDAV 端到端加密同步

- 自托管 Web、Windows 和 macOS 新增用户自有 WebDAV 同步；使用独立随机 256 位同步密钥，不复用主密码或 WebDAV 密码。
- 新增 Sync Bundle V1：紧凑 JSON 经 Gzip 后使用 AES-256-GCM 加密，AAD 绑定对象类型、Vault UUID 和对象 ID；远端 head 与完整快照均为密文。
- 远端提交要求 HTTPS、标准强 ETag、`If-None-Match` 和 `If-Match`，先上传不可变快照再条件推进 head；删除远端也使用 head ETag 条件保护，拒绝无条件覆盖或并发删除。
- 支持解锁、Vault 写入后 5 秒防抖、后台超过 60 秒返回和手动同步；同步接口自身不会递归触发下一次同步。
- 新增完整 Vault 三方合并；同条目、标签、密码组或扩展字段双端修改时逐项选择本机、远端或条目保留两份，删除/修改冲突不会静默删除。
- 新增最近 10 个端到端加密历史版本、恢复为新的最新版本、断开本机、双确认删除远端和同步密钥轮换。
- 新增主密码二次验证后的二维码、配对 URI 和分段恢复码；创建同步空间不再直接返回同步密钥材料。

### UI、兼容性与安全

- 顶部增加紧凑同步状态按钮，“设置 > 同步”提供连接、自动同步、历史、配对和危险操作；配置、冲突和恢复码使用独立宽版响应式弹窗。
- 冲突 API 与界面只显示标题、状态、时间、字段数量和变化区域，不包含字段值；锁定时清除 WebDAV 密码、主密码、恢复码和二维码内存状态。
- 本机 `sync-settings.enc` 和 `sync-base.enc` 使用 Vault 用途隔离密钥加密；修改主密码时一起重加密，导入不同 `vault_id` 时自动停用旧同步。
- Vault V1 新增可选稳定 `vault_id`；旧 Vault 缺省时继续缺省，不会被写入 `vault_id: null`。Android 新建 Vault 开始生成 UUID，但 V5.2 不开放移动同步 UI。
- WebDAV `401/403` 映射为同步上游错误，不会误触发 SecretBase 会话锁定；能力探测会验证过期 ETag 并清理临时目录。
- WebDAV 下载使用 64 MiB 流式硬限制；生产接口不允许本机回环 HTTP，自动化测试通过注入本机传输适配器完成。
- 主密码轮换和导入不同 Vault 的安全文件更新使用逆序回滚事务；任一同步配置/基线写入失败都不会留下半清理状态。
- 待处理加入冲突不再伪装成已配置连接；旧解锁会话的迟到响应不会覆盖新会话状态，自动同步遇到短暂忙碌会重新排队。
- 密钥轮换在本机新配置落盘后才清理旧历史，失败回退会恢复旧密钥、原历史链和本机基线。
- 同步服务拆分为协议加密、WebDAV 传输、远端仓库、合并、运行态、核心同步和管理模块，避免单文件继续增长。
- 新增 Python/Rust Sync Bundle V1 公开黄金向量、内存 WebDAV 端到端回归、前端自动触发/锁定清理测试和发布模块体量门禁。

### 发布边界

- V5.2 首版同步只覆盖自托管 Web、Windows 和 macOS；Android 继续保持本地 Vault 模式，V5.3 将直接复用同一 Rust 协议。
- 不提供官方云、账号系统、团队共享、实时推送、忽略 TLS 证书错误或弱 ETag 兼容模式。
- Vault envelope、数据目录、AI 隐私黑名单、自动填充绑定、生物识别凭据和签名更新基线保持不变。

## 5.1.3 - 2026-07-17

### Android 界面稳定性

- 修复连续多次切到后台并通过指纹返回后，软键盘 Insets 可能残留，导致主内容只显示上半屏、底部出现大面积空白的问题。
- 已锁定的密码库不再自动聚焦主密码输入框；首次创建密码库仍保留自动聚焦，避免影响初始化体验。
- 进入后台、设备锁定、发起指纹认证和认证结束时都会清除输入焦点并隐藏软键盘；Android 原生桥会重新请求窗口 Insets 和布局。
- 新增锁定页焦点与生命周期键盘回归测试，并通过 Android Kotlin Release 编译和完整 Flutter 测试。
- 本版本不修改 Vault V1、默认数据目录、Android 永久签名、自动填充数据或 AI 隐私边界。

## 5.1.2 - 2026-07-17

### Android 安全与更新

- 更新检查增加三次有限重试，清单连接超时提升至 20 秒；临时检查失败不再错误写入 24 小时成功检查时间。
- APK 建连超时提升至 60 秒、下载流空闲超时提升至 120 秒，并支持受严格 `Content-Range` 校验的断点续传。
- 断网、超时和用户暂停后保留 `.part` 下载进度；服务器忽略 Range 时自动完整覆盖，返回 403/416 或非法范围时清理部分文件并有限重试。
- 续传完成后继续校验文件大小、SHA-256、包名、`versionCode` 和永久签名；设置页显示已下载大小，并提供继续下载与重新下载入口。
- 应用真正进入后台后，返回时立即锁定并进入指纹或主密码认证；系统指纹弹窗造成的短暂失焦不会触发循环锁定。
- Android Keystore 临时不可用、设备尚未完成解锁或一次认证过期时不再删除指纹凭据；仅在密钥永久失效、凭据损坏或 Vault 绑定失配时要求重新开启。

### 体积与发布

- 正式 Release 在通用 APK 之外新增 `armeabi-v7a`、`arm64-v8a` 和 `x86_64` 独立 APK；实测大小分别约 21.1 MB、24.1 MB 和 25.8 MB。
- V5.1.2 客户端会从签名更新清单中优先选择设备匹配的 ABI 包，并在缺失时回退通用包。
- APK 门禁同时检查每个目标 ABI 的 Flutter 引擎、AOT 应用库和 Rust Vault 库，避免产出仅包含部分运行时的错误包。
- 本版本不修改 Vault V1、默认数据目录、永久 Android 签名、更新签名密钥或 AI 隐私边界。

## 5.1.1 - 2026-07-17

### Android 更新稳定性

- 确认 V5.0.0 至 V5.0.2 默认开启自动预下载时，会因缺少网络状态权限而在发现新版本后显示“检查更新失败”；V5.1.0 已补齐权限，V5.1.1 进一步增加异常降级和正式 APK 权限门禁，旧版可关闭自动预下载后手动升级。
- 发现新版本后即使读取网络类型失败，也会保留版本信息和“下载更新”入口，不再覆盖为检查失败；离线状态会明确提示联网后手动下载。
- 下载失败、取消下载或系统安装界面启动失败后保留重试入口；打开 Android 系统安装界面后立即退出处理中状态，用户取消安装返回时不会卡死。
- 网络、DNS、VPN、TLS 证书、系统时间和 HTTP 状态使用可操作的中文错误提示，不再统一显示模糊失败信息。
- 新增更新控制器回归，覆盖网络状态异常、下载失败、安装启动失败和取消系统安装后的可重试状态。

### 发布可靠性

- Android APK 校验新增 `INTERNET`、`ACCESS_NETWORK_STATE` 和安装权限契约，防止发布包再次遗漏更新所需权限。
- macOS DMG 创建、挂载和卸载统一使用三次有限重试，保留 `hdiutil` 原始诊断输出；构建任务和下载包复测共用同一实现。
- 本版本不修改 Vault V1、默认数据目录、永久 Android 签名、更新清单密钥或 AI 隐私边界。

## 5.1.0 - 2026-07-17

### Android 系统自动填充

- 新增 Android 10+ 原生 `AutofillService`，可在系统设置中将 SecretBase 设为默认自动填充服务；Android 11+ 支持输入法行内建议。
- 自动填充必须先通过指纹或主密码验证，再由用户明确选择候选条目；候选阶段不返回密码值，最终选择前不会把密码放入 Intent、日志或网络请求。
- 支持按网站域名排序候选、手动确认账号/密码字段、记住目标与条目绑定，以及在设置中管理保存提示、行内建议和已停用目标。
- 新增登录后保存提示；注册和修改密码页面不会填入已有密码，但可在用户提交后确认保存新登录信息。分步登录会在同一目标的多个系统上下文中合并账号信息。
- 自动填充绑定和字段映射使用 Vault 会话派生密钥写入独立的 `autofill-settings.vault`；修改主密码时重新加密，创建、导入或替换 Vault 时清除无效绑定。
- 原生自动填充会话限制为 120 秒且最多保留 3 个；新登录信息仅在进程内短暂保存，并继续通过 Rust Vault 事务、原子写入和恢复副本落盘。

### 交互与维护

- Web、桌面和 Android 从密码组条目页新建条目时，无论组内是否已有内容，都会默认选中当前密码组；标签来源采用相同的上下文继承规则。
- Android 设置页只在 SecretBase 本身被系统选中时显示“已启用”，不再把其他密码管理器误判为 SecretBase；设置更新失败会给出明确反馈。
- APK 门禁新增 Autofill Service、原生认证 Activity、JNI 导出、R8 保留和模拟器服务注册检查；Android Release Lint 保持零错误。
- 本版本不修改 Vault V1、默认数据目录、发布签名或 AI 隐私边界；iOS 继续搁置。

## 5.0.2 - 2026-07-16

### 修复与交互

- Android AI 管家支持与 Web/桌面一致的跨密码组、标签、条目结构和条目创建复合计划；计划按确定顺序原子执行，存在删除/分配、重复治理或字段来源冲突时由本机拒绝。
- AI 返回的 `domain` 与实际动作不一致时保留可审核计划并显示归类警告，不再丢失整轮建议。
- Web 默认搜索标题、网址、标签、字段名和备注；非隐藏字段值仍需用户主动选择，避免空搜索范围导致“输入后没有结果”。
- 浏览器禁用站点存储时，认证 token 保留在当前内存会话，分页、侧栏和常用筛选偏好安全降级，不再阻止应用初始化。
- 代理或服务返回非 JSON 错误页时保留真实 HTTP 状态，401 仍触发锁定，502 不再误报为普通网络断开。
- 条目、密码组和标签编辑器补齐长度限制；空白标题和字段名在发送前处理，减少保存后才出现验证错误的情况。

### 部署、安全与维护

- Nginx 对全部 `/api/ai/` 接口统一使用 180 秒代理超时，覆盖对话提交、诊断和后续新增 AI 能力。
- 前端缓存参数统一使用应用版本号，模板加载器复用运行时版本；桌面入口页和运行时配置使用 `no-store`，避免升级后继续加载旧界面。
- 后端日志不再记录条目标题、标签名或密码组名，只保留操作类型和数量。
- 后端为 AI 录入/计划、批量 ID、条目字段、分类数量和排序请求增加边界，首次主密码与移动端统一为 8 至 128 个字符，并统一清理标题、URL 与字段名的首尾空格。
- 修复 Vault Core 默认 `cargo test --locked` 因测试 feature 门控而编译失败的问题；CI 同时验证默认命令和全 feature 黄金向量。
- 升级 `cryptography`、`python-multipart` 和移动端传递依赖 `tokio` 至已修复版本，清除本次发布前的已知依赖漏洞。
- 桌面依赖清单只保留桌面专属包，构建入口显式同时安装后端清单，避免重复依赖图和重复安全告警。
- 删除前端测试中分散的缓存时间戳硬编码，版本一致性改为从 `backend/version.py` 动态校验。

### 安全与兼容性

- Vault 和加密 AI 配置保存在 Android 私有目录；禁止系统备份、明文网络流量和敏感页面截图。

- AI 默认只允许 HTTPS，首次发送要求隐私同意，每次应用建议前必须预览、勾选并校验 revision。
- 普通 AI 只发送 hostname 和结构元数据，并使用临时别名；已有字段值、完整 URL、备注和真实 UUID 不进入网络请求。
- AI 新建是唯一允许主动发送新字段值的模式，必须二次确认；敏感原文不写入对话历史或后续上下文。
- 模型协议和执行器拒绝删除条目、字段、字段值和密码组，也拒绝修改已有 URL/备注；所有 AI 写入前创建加密恢复快照。
- 对话管家和原五项专业工具统一改用服务端不可篡改计划令牌，客户端只能提交选中项 ID；发布门禁新增后端 AI 安全测试和前端对话契约测试。
- 拆分 Web AI 状态、领域装配、厂商设置控制器和工作区样式，并拆分 Android AI 响应校验与管家组件，修复本地即时回复卡在处理中及部分厂商结构化参数不兼容的问题。
- Web AI 管家改为无提示词预检、逐次确认后发送；确认令牌、计划令牌和撤销令牌均原子消费，修复自动跳过确认、重复调用模型和确认界面卡住的问题。
- 专业工具与 AI 服务设置改为 AI 管家上层面板，修复遮挡后无法交互的问题；统一侧栏入口对齐，并重做居中限宽输入器、发送确认、乐观消息、失败恢复和窄屏布局。
- 修复 Web AI 输入栏继承全宽主按钮后挤压模式、范围和隐私提示的问题，并收紧未配置状态的服务设置按钮。
- Web AI 对话历史在桌面端改为默认展开的可收缩左侧栏，收起后保留窄侧轨；窄屏默认收起并支持点击遮罩或选择会话后关闭。

- Web/桌面 AI 管家支持跨密码组、标签、条目结构和条目创建的原子复合计划：审核页按领域分组，一次确认、一次快照、一次撤销；高影响操作默认不选，服务端会阻止存在顺序歧义的选中组合。
- Android 移动界面完成 Material 3 交互升级：统一页面标题与数量层级，强化底部导航选中态，压缩条目、密码组和标签卡片，并将卡片编辑/删除收纳到大触控面积的底部操作面板。
- Android AI 管家将快捷整理、模式、范围和专业工具收纳到输入框旁的圆形 `+` 面板，保留紧凑上下文提示、底部发送确认和按领域折叠审核；AI 请求处理中仍可查看工具与设置，但不能发起冲突请求。
- Web/桌面密码组卡片和详情页补齐删除入口，删除只解除条目归属关系；组内清除筛选会返回密码组视图，并在删除后自动校正分页。
- Android 密码组与标签卡片使用更明确的“编辑/删除”入口，底部操作面板适配矮屏；从密码组或标签进入条目后，返回和清除会回到来源页而不是全部条目。
- 修复 Web/桌面 Store 与 Vue 页面状态分离导致密码组删除、标签新增/编辑/删除后界面不立即更新；写操作不再隐式重复加载，由控制器统一刷新可见数据并通过运行时契约测试保护。

- 修复“生成密码组管理计划”被本地意图判断误识别为生成真实密码的问题，同时保留密码组内本地生成密码的能力。
- AI 服务返回普通文本或无效 JSON 时不再触发 500；对话模式保留文字回复并禁止生成操作，AI 新建模式则返回受控错误且不创建内容。

- 更新清单使用独立 Ed25519 密钥签名；安装包下载后复核大小与 SHA-256，Android 还会核对包名、版本号和永久发布签名。
- 普通 Android CI 继续使用一次性测试密钥，只有受保护的 GitHub `release` 环境可以访问永久 JKS 和更新签名私钥。
