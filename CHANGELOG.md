# 变更记录

## 5.0.0 - 开发中

### 新增

- 新增 Android 10+ Flutter + Rust 离线客户端，不启动浏览器、Python、FastAPI 或本地 HTTP 服务。
- 新增 Vault 会话、原子持久化、revision 冲突、恢复副本、主密码修改和加密导入导出。
- 新增条目、标签、密码组、回收站、自定义字段、分页、搜索和模板复制的移动端体验。
- 新增五项需确认的 AI 能力：文本解析、单条目标签整理、密码组整理、标签治理和自然语言操作计划。
- 新增 Web/桌面与 Android 默认对话式 AI 管家、加密会话历史、全部/指定范围、条目定位和本地计划审核。
- 新增 Android 强生物识别解锁：主密码不会保存，设备专属 Vault 解锁密钥由 Android Keystore 加密并要求指纹放行。
- 新增 Android AI 一次性撤回，应用与撤回均校验 Vault revision、创建加密恢复副本并同步刷新条目与分类。
- 新增 Android 根页面连续返回两次退出，确认退出前先锁定 Vault；弹窗、来源筛选、子页面和主标签返回优先于退出提示。
- 新增 OpenAI、DeepSeek、Kimi、智谱 GLM、SiliconFlow、Gemini、OpenRouter 及自定义 OpenAI-compatible 预设，Base URL 可编辑，模型 ID 可手工填写。
- 新增三 ABI Android CI、API 29/36 模拟器冒烟测试、APK 隐私扫描和可选持久签名流程。

### 安全与兼容性

- Vault V1 保持兼容，旧条目缺失默认字段时安全补全，未知根、条目和字段扩展继续保留。
- Vault 和加密 AI 配置保存在 Android 私有目录；禁止系统备份、明文网络流量和敏感页面截图。
- 隐藏字段在 Rust 列表接口中先行掩码，敏感剪贴板内容使用 Android 敏感标记并自动清理。
- AI 默认只允许 HTTPS，首次发送要求隐私同意，每次应用建议前必须预览、勾选并校验 revision。
- 普通 AI 只发送 hostname 和结构元数据，并使用临时别名；已有字段值、完整 URL、备注和真实 UUID 不进入网络请求。
- AI 新建是唯一允许主动发送新字段值的模式，必须二次确认；敏感原文不写入对话历史或后续上下文。
- 模型协议和执行器拒绝删除条目、字段、字段值和密码组，也拒绝修改已有 URL/备注；所有 AI 写入前创建加密恢复快照。
- 对话管家和原五项专业工具统一改用服务端不可篡改计划令牌，客户端只能提交选中项 ID；发布门禁新增后端 AI 安全测试和前端对话契约测试。
- 拆分 Web AI 状态、领域装配、厂商设置控制器和工作区样式，并拆分 Android AI 响应校验与管家组件，修复本地即时回复卡在处理中及部分厂商结构化参数不兼容的问题。
- Web AI 管家改为无提示词预检、逐次确认后发送；确认令牌、计划令牌和撤销令牌均原子消费，修复自动跳过确认、重复调用模型和确认界面卡住的问题。
- 专业工具与 AI 服务设置改为 AI 管家上层面板，修复遮挡后无法交互的问题；统一侧栏入口对齐，并重做居中限宽输入器、发送确认、乐观消息、失败恢复和窄屏布局。
- 修复 Web AI 输入栏继承全宽主按钮后挤压模式、范围和隐私提示的问题，并收紧未配置状态的服务设置按钮。
- Web AI 对话历史在桌面端改为默认展开的可收缩左侧栏，收起后保留窄侧轨；窄屏默认收起并支持点击遮罩或选择会话后关闭。
- Web/桌面 AI 管家将常用整理指令固定到输入框上方；计划类型声明错误时自动按实际动作归类，并修复确认与专业工具按钮换行。
- AI 发送过程中仍可打开专业工具和服务设置且不会取消或重发请求；计划项可在独立详情层按页查看本机完整条目，隐藏字段默认遮罩，完整内容不会补发给 AI。
- Web/桌面 AI 管家支持跨密码组、标签、条目结构和条目创建的原子复合计划：审核页按领域分组，一次确认、一次快照、一次撤销；高影响操作默认不选，服务端会阻止存在顺序歧义的选中组合。
- Android 移动界面完成 Material 3 交互升级：统一页面标题与数量层级，强化底部导航选中态，压缩条目、密码组和标签卡片，并将卡片编辑/删除收纳到大触控面积的底部操作面板。
- Android AI 管家将快捷整理、模式、范围和专业工具收纳到输入框旁的圆形 `+` 面板，保留紧凑上下文提示、底部发送确认和按领域折叠审核；AI 请求处理中仍可查看工具与设置，但不能发起冲突请求。
- Web/桌面密码组卡片和详情页补齐删除入口，删除只解除条目归属关系；组内清除筛选会返回密码组视图，并在删除后自动校正分页。
- Android 密码组与标签卡片使用更明确的“编辑/删除”入口，底部操作面板适配矮屏；从密码组或标签进入条目后，返回和清除会回到来源页而不是全部条目。
- 修复 Web/桌面 Store 与 Vue 页面状态分离导致密码组删除、标签新增/编辑/删除后界面不立即更新；写操作不再隐式重复加载，由控制器统一刷新可见数据并通过运行时契约测试保护。
- 修复 Android 重复锁屏事件或长时间后台恢复后“内容已保护”遮罩不释放，以及桌面端托盘隐藏暂停动画帧后保护层可能持续覆盖解锁页的问题。
- Web/桌面的确认、条目、密码组、标签、标签合并和批量归组操作增加单次提交锁与处理反馈；Android 保存、恢复、分类排序、AI 设置和条目变更期间禁止误关弹窗并显示进度。
- 修复“生成密码组管理计划”被本地意图判断误识别为生成真实密码的问题，同时保留密码组内本地生成密码的能力。
- AI 服务返回普通文本或无效 JSON 时不再触发 500；对话模式保留文字回复并禁止生成操作，AI 新建模式则返回受控错误且不创建内容。
- AI 服务设置新增真实模型兼容性诊断：经用户确认后，用 16 组合成数据覆盖计划生成、混合任务、越权操作、提示注入、多轮和长上下文，并展示通过、降级、拦截及失败报告。
- Rust 构建路径会重映射为通用虚拟路径，APK 验证拒绝真实工作区、用户目录和私人域名。

## 4.0.0 - 未发布

### 新增

- 固化 SecretBase Vault V1 二进制封装、JSON payload 和错误分类规范。
- 新增公开黄金向量，覆盖 UTF-8、隐藏字段、标签、密码组、回收站和未知字段。
- 新增独立 Rust 参考核心，实现内存中的头部检查、payload 校验、加密和解密。
- CI 与正式 Release 增加固定 Rust 工具链、格式、Clippy 和跨语言兼容门禁。

### 兼容性与安全

- Vault envelope 和 payload 仍保持 V1，不迁移现有用户文件。
- Python 继续作为 Web 与桌面生产核心；Rust 原型暂不进入应用包或运行链路。
- 根文档、条目和自定义字段中的未知 JSON 字段在读取和写回时保留。
- 错误密码和密文损坏继续统一为认证失败，不暴露可用于密码探测的差异。

## 3.3.0 - 2026-07-12

### 新增

- 新增 macOS 13+ Apple Silicon arm64 独立桌面应用构建，提供 DMG 和 ZIP。
- 新增 Windows/macOS 共用的桌面平台能力接口、诊断信息和统一发布流水线。

- macOS 首版关闭窗口时退出本地服务，不提供菜单栏驻留；删除 `.app` 默认保留应用支持目录中的数据。
- Windows 与 macOS 继续使用相同 vault 格式，并通过加密备份迁移数据。

## 3.2.0 - 2026-07-11

### 新增

- 新增当前用户免管理员 Windows 安装器，并继续保留便携 ZIP。
- 新增桌面状态与诊断、数据目录快捷入口和手动更新检查。
- 新增可选系统托盘；关闭到托盘前会锁定 vault 并清除会话。
- 新增关闭确认，可选择隐藏到托盘、完全退出或取消，并可记住选择、不再提醒。
- 新增 WebView2 原生 `Ctrl + 滚轮` 界面缩放，缩放时在窗口顶部显示当前比例。
- 新增可选 Windows 代码签名流水线，未配置证书时仍可构建未签名产物。

### 安全与兼容性

- 安装、升级和默认卸载均保留 `%LOCALAPPDATA%\SecretBase` 中的用户数据。
- 只有显式勾选并输入 `DELETE` 后，卸载器才删除 vault、备份、设置、日志和 WebView 数据。
- 手动更新检查只查询 GitHub 正式 Release，不发送 vault 内容或其他敏感数据。
- 托盘隐藏和托盘“锁定”会立即清空前端敏感状态，恢复窗口时直接显示解锁页。
- 桌面窗口最小尺寸降至 `360 × 320`，可像浏览器一样缩放到窄窗口并继续滚动操作。
- 保存关闭偏好不再立即创建托盘线程，修复设置关闭到托盘时跳转解锁页并卡住的问题。
- 安装、升级和卸载会请求已有桌面实例优雅退出，不再被“关闭到托盘”拦截。
- 保持服务端部署、V3.1 便携数据目录和 vault 文件格式兼容。

## 3.1.0 - 2026-07-11

### 新增

- 新增基于 pywebview 与 Edge WebView2 的 Windows 独立桌面窗口。
- 新增单实例运行，重复启动时恢复并聚焦已有窗口。
- 新增 Windows 原生“另存为”导出桥，桌面导出不再依赖浏览器下载行为。
- 新增 PyInstaller one-folder 构建、自检、敏感文件扫描和 SHA-256 校验流程。

### 兼容性

- 保留现有服务端部署、源码启动器和 vault 文件格式。
- Windows 桌面包使用独立本地数据目录，不在发布目录写入用户数据。
- 固定 `pythonnet 3.0.5` 与 `clr-loader 0.2.10`，避免新依赖版本导致 WinForms 运行时无法加载。
- 随 EXE 提供 .NET 应用配置，允许 Chrome 等浏览器下载的 ZIP 在继承 Mark-of-the-Web 后正常加载 `Python.Runtime.dll`。
- Windows CI 增加 CLR、WinForms、Edge WebView2 和模拟 Internet Zone 标记的打包运行时自检。

## 3.0.0 - 2026-07-10

### 新增

- 桌面基础模式：隔离本地数据路径并使用随机回环端口。
- Windows、Linux 和 macOS 源码环境的一键本地启动。
- 本地 vendored Vue 运行时，支持前端离线启动。
- Linux/Windows CI 和由 Git 标签触发的 GitHub Release 自动化。
- 密码组、完整标签管理、AI 整理和需确认的 AI 操作计划。

### 安全

- 桌面 CORS 和 Host 只接受当前本地应用来源。
- 防止认证表单退化为携带凭据的 GET 请求。
- 移除 vault 页面中的第三方 favicon 和字体请求。
- 增加浏览器安全头，以及外部链接的 opener/referrer 隔离。

### 调整

- API 和桌面运行信息统一使用 `3.0.0` 版本常量。
- 固定 Python 直接依赖版本，提高本地启动可复现性。
- 服务端偏好设置默认保存在可写数据目录，并补充旧版 `backend/settings.json` 的升级说明。
- V3.0 文档从待实施规划更新为已完成的桌面基础模式说明。

### 兼容性

- Vault 文件格式保持 `1.0` 不变。
- 现有服务器部署和加密 vault 文件继续兼容。
