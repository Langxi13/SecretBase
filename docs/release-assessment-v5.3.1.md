# SecretBase V5.3.1 发布评估

## 结论

V5.3.1 是 V5.3 的交互维护版本，补齐了跨设备同步的配对接收链路，不改变 V2 加密快照格式、Vault V1 数据格式或 WebDAV 存储协议。

## 本次补齐

- Android 可直接扫描 Web/桌面端显示的 SecretBase V2 配对二维码。
- Android 可粘贴配对链接，或从系统剪贴板读取后自动填充 WebDAV 地址、用户名和恢复码。
- Web/Windows/macOS 加入页可粘贴 `secretbase://sync/join` 配对链接，并识别 V1/V2 恢复码。
- Web/Windows/macOS 会在本地校验恢复码校验和，并兼容 V5.3.0 旧版 `key=` 配对链接后转换为恢复码。
- Android 加入页提供独立 WebDAV 连接测试；测试只做能力探测，不写入同步空间。
- Android 可安全复制恢复码和配对链接，使用本机剪贴板自动清理策略。
- 新生成的配对链接只携带恢复码、空间元数据、WebDAV 地址和用户名，不携带 WebDAV 应用密码。
- Android 保留对 V5.3.0 旧 V2 配对 URI 的兼容转换；V1 配对链接仍由 Web/桌面端处理，Android 同步只支持 V2。
- Vault 锁定时清除 Android 同步弹窗中的密码、恢复码、配对材料和冲突上下文。

## 安全边界

- 二维码、配对链接和恢复码都是同步密钥材料，只能在再次验证主密码后显示或复制。
- WebDAV 应用密码不会进入二维码、配对链接、恢复码、日志、AI 或远端快照。
- 扫描器只接受 `secretbase://sync/join`，拒绝 HTTP 链接、带 URL 内嵌凭据的地址和未知协议。
- 扫描结果不会自动发起同步；用户仍需确认填写内容并手动输入 WebDAV 应用密码后点击“加入并同步”。
- 旧链接中的裸密钥只在当前客户端内存中转换为恢复码，不写入持久化存储；新生成的链接不会再携带裸 `key`。
- Android 从剪贴板读取配对链接后，会在内容仍匹配时清空该剪贴板；扫码和手动输入不会自动发起同步。

## 验证范围

- Python 同步 V1/V2 恢复码和配对 URI 测试。
- 前端配对链接解析、协议识别和敏感状态清理测试。
- Flutter 配对 URI 往返、旧 V2 链接转换、危险 URL 拒绝测试。
- Flutter Analyze、完整 Flutter 测试、Android Release Manifest 摄像头权限和 APK 结构门禁。
- 后端、前端、桌面、AI 和 Rust 同步全量发布检查。

## 已知限制

- Android 扫码需要用户授予摄像头权限；拒绝权限时仍可手动粘贴恢复码或配对链接。
- WebDAV 应用密码不进入二维码，因此扫码后仍必须手动输入。
- Android 当前只加入 V2 快照空间，不自动迁移旧 V1 空间。
- iOS 继续搁置，不属于本次维护范围。
