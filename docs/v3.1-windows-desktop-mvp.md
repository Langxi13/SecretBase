# SecretBase V3.1 Windows 桌面 MVP 规划

本文档记录 V3.1 的暂定计划。V3.1 的目标是交付第一个可用的 Windows 桌面便携版，而不是完整安装器或产品化桌面套件。

## 1. 版本目标

V3.1 交付一个 Windows one-folder 便携包，用户可以双击 `SecretBase.exe` 打开桌面窗口并离线使用 SecretBase。

成功后的状态应是：

- Windows 上可以构建 `dist/SecretBase/` 目录包。
- 目录包内包含 `SecretBase.exe`、后端运行依赖、静态前端和必要桌面资源。
- 双击 exe 后打开 SecretBase 桌面窗口，而不是默认浏览器。
- 桌面窗口关闭时，本地后端随之退出。
- 应用继续使用桌面数据目录保存 vault、备份、日志和设置。
- 便携包不包含真实 vault、日志、`.env`、备份文件或用户数据。

V3.1 仍不做安装器、代码签名、自动更新、系统托盘、开机启动、浏览器自动填充、云同步或手机端能力。

## 2. 产物形态

V3.1 采用 PyInstaller one-folder 目录包，而不是单文件 exe。

暂定产物：

```text
dist/
└── SecretBase/
    ├── SecretBase.exe
    ├── _internal/
    └── 其他 PyInstaller 运行依赖
```

选择目录包的原因：

- 更容易定位依赖和资源缺失问题。
- pywebview、FastAPI、uvicorn 和静态前端资源更容易打包和排错。
- 后续如果要做安装器，可以直接把目录包作为安装源。

V3.1 不追求单文件 exe。单文件模式启动更慢，资源解包和故障排查成本更高，放到后续版本再评估。

## 3. 桌面窗口壳

V3.1 新增基础 pywebview 桌面壳。

暂定入口：

```text
desktop/app.py
```

职责：

- 设置 `SECRETBASE_MODE=desktop`。
- 解析桌面数据根目录。
- 分配本机随机端口。
- 启动本地 FastAPI 后端。
- 等待 `/health` 可用。
- 用 pywebview 打开 `http://127.0.0.1:<port>/`。
- 设置窗口标题为 `SecretBase`。
- 设置合理的默认窗口尺寸。
- 窗口关闭时停止后端并退出应用。

V3.1 的桌面壳只提供基础窗口，不新增菜单栏、托盘、关于页、启动页、窗口级错误页或数据目录快捷入口。

## 4. 后端启动策略

V3.0 的 `desktop/launcher.py` 已经证明同源桌面模式可用。V3.1 应继续复用这套运行模式，但桌面包内不建议再通过 PyInstaller 包里的 exe 递归拉起子进程。

暂定方案：

- 桌面 app 入口在同一进程内用后台线程启动 uvicorn/FastAPI。
- 后端继续监听 `127.0.0.1:<动态端口>`。
- 前端仍由 FastAPI 在 `desktop` 模式下同源托管。
- 关闭 pywebview 窗口时触发后端停止逻辑。
- debug 模式可以保留控制台或日志输出，普通双击启动默认隐藏控制台。

这样可以减少打包后进程路径、解释器路径和子进程资源定位问题。

## 5. AI 配置策略

AI 配置改为解锁后在设置页完成，桌面用户不需要修改启动环境。V3.1 复用当前设置页 AI 页签，不为桌面包单独新增 AI 配置界面。本阶段只支持一个当前启用的 OpenAI-compatible provider，不引入多 provider 管理。

暂定原则：

- 设置页提供“AI 配置”页签。
- 配置项包括 Base URL、API Key 和从服务商实时返回列表中选择的模型。
- Base URL 示例包括 `https://api.deepseek.com` 和 `https://api.openai.com/v1`，不提供固定默认 provider。
- 配置设计为 OpenAI-compatible 通用接口，不做 DeepSeek 专用绑定。
- 保存前必须通过 `<base_url>/models` 拉取模型列表，并用所选模型执行固定无敏感连通测试。
- API Key 使用当前解锁 vault 的用途派生密钥加密后保存。
- API Key 不写入明文 `settings.json`。
- API Key 不进入 vault 备份和恢复。
- 换机器或迁移 vault 后，需要重新配置 AI Key。
- 未配置 AI 时，核心密码库功能不受影响。

暂定安全设置文件：

```text
<桌面数据根目录>/secure-settings.enc
```

该文件只保存本机敏感配置，例如 AI API Key。它不是 vault 数据，不随加密备份迁移。

设置接口应避免返回完整 API Key。前端展示“已配置/未配置”、当前 Base URL、模型和 Key 掩码；只有点击“修改配置”才进入编辑态。如果需要修改 Key，用户重新输入即可。清除 Key 应是显式动作。

## 6. 打包和构建

V3.1 Windows 包在 Windows 本机上构建。

原因：

- PyInstaller 不是跨平台交叉编译工具，Windows 包应在 Windows 环境运行 PyInstaller。
- pywebview 的 Windows 运行依赖需要在 Windows 上验证。
- 当前 Linux 环境可用于准备代码、文档和大部分自动化测试，但不作为 Windows exe 最终验收环境。

暂定新增内容：

- `desktop/requirements.txt`：桌面打包依赖，包括后端依赖、`pywebview`、`pyinstaller`。
- `scripts/build-desktop-windows.ps1`：Windows 构建脚本。
- PyInstaller spec 文件：固定入口、资源、图标、版本信息和排除项。
- 基础应用图标和版本信息。

构建脚本必须明确排除：

- `backend/.env`
- `backend/data/`
- `backend/logs/`
- `backend/settings.json`
- vault 文件
- 备份文件
- 测试脚本
- 文档截图
- `.git/`

## 7. 基础品牌

V3.1 只做基础品牌，不做完整视觉系统。

包含：

- 应用名：`SecretBase`
- 窗口标题：`SecretBase`
- 基础 `.ico` 图标
- exe 版本信息

不包含：

- 启动页
- 关于页
- 多尺寸完整品牌套件
- 安装器图标体系
- 产品官网式展示页

## 8. 验收标准

### 自动化验证

V3.1 代码合入前应通过：

```bash
python -m compileall backend desktop scripts
node --check frontend/js/app.js frontend/js/api.js frontend/js/store.js frontend/js/utils.js
venv/bin/python scripts/test-desktop-foundation.py
venv/bin/python scripts/test-backup-separation.py
venv/bin/python scripts/v1-fake-smoke-test.py
```

AI 配置相关测试：

- 保存 AI API Key 后，明文 `settings.json` 不包含 API Key。
- 加密安全设置文件不包含 API Key 明文。
- 解锁后 AI 路由可以读取加密保存的配置。
- 锁定状态下设置接口仍返回 401。
- 清除 API Key 后，AI 状态正确变为未配置。
- 旧设置文件没有 AI 字段时仍可正常加载。

### Windows 手动验收

在 Windows 本机执行：

1. 创建干净虚拟环境。
2. 安装桌面依赖。
3. 运行构建脚本。
4. 双击 `dist/SecretBase/SecretBase.exe`。
5. 确认桌面窗口打开，且默认不显示额外控制台。
6. 初始化主密码。
7. 解锁 vault。
8. 创建、编辑、删除条目。
9. 使用搜索、标签、回收站。
10. 创建手动备份并查看备份中心。
11. 测试加密备份导入导出。
12. 在设置页配置 AI，确认 AI 状态变为已配置。
13. 关闭窗口后确认本地后端进程退出。
14. 检查便携包内不包含真实数据、`.env`、日志或备份。

## 9. 暂不进入 V3.1 的内容

以下内容不进入 V3.1：

- Windows 安装器。
- 自动更新。
- 代码签名。
- 系统托盘。
- 开机启动。
- 浏览器插件或自动填充。
- 云同步。
- 手机 App。
- 保存或自动填充主密码。
- 完整品牌系统。
- 单文件 exe。

这些内容继续放到 V3.2 或后续版本讨论。

## 10. 后续讨论重点

下一轮可继续讨论桌面打包和窗口壳细节。AI 配置当前不继续扩展多 provider，也不新增模型能力；后续如果要支持多个 provider，应作为独立产品需求重新评估设置结构、默认 provider、迁移策略和安全测试范围。
