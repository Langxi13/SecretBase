# SecretBase App 化长期规划

本文档记录 SecretBase 从自托管 Web 工具逐步演进为桌面应用和手机应用的长期方向。它不是具体开发任务清单，而是用于后续版本需求讨论、边界确认和实施计划拆分的产品路线基准。

## 1. 长期目标

SecretBase 的长期目标是成为一个以个人本地数据安全为核心的离线密码库：

- 可以继续作为自托管 Web 服务运行。
- 可以打包为 Windows 桌面应用，优先满足完全本地离线使用。
- 未来可以打包为手机 App，使用同一套 vault 数据规则和加密规则。
- 数据迁移优先依赖手动加密备份文件，而不是云同步或账号系统。

V3.0 桌面基础模式、V3.1 Windows MVP、V3.2 Windows 产品化、V3.3 macOS arm64 和 V4.0 共享 vault 核心准备均已完成。V5.0 Android 核心实现和本机 arm64 APK 构建已经完成，当前处于 CI、模拟器、真机和跨端备份验收阶段。

## 2. 当前基线

当前 SecretBase 的基础形态如下：

- 后端使用 FastAPI，负责认证、vault 读写、条目管理、标签、回收站、备份、导入导出、工具报告和 AI 解析。
- 前端使用本地 vendored Vue 3 global build、原生 JavaScript 和原生 CSS，无 npm 构建链。
- 数据以本地加密 vault 文件保存，不依赖数据库。
- 已具备手动备份、自动备份、备份中心、导入导出和三步恢复流程。
- 当前部署模型仍以自托管 Web 为主，推荐在 nginx、HTTPS 和额外访问控制之后运行。
- `v3.0.0` 已支持 Windows、Linux 和 macOS 源码环境的一键本地启动，前端运行时不依赖外部 CDN。

这些能力会作为 App 化的基础复用。短期内不改变 vault 文件格式，不重写核心密码库能力。

## 3. 总体路线

App 化路线采用渐进式演进：

1. 先支持桌面本地运行模式。
2. 再完成 Windows 桌面离线包。
3. 然后补齐桌面产品化体验。
4. 为手机 App 固化共享 vault 格式和跨语言核心。
5. 以 Android 为首个平台实现完全离线的手机 App，再适配 iOS。

桌面首版选择 `PyInstaller + pywebview`，原因是它能最大程度复用现有 Python 后端和静态前端，首版改造面较小。手机端不计划内嵌 Python/FastAPI，长期应通过共享 vault 核心实现离线能力。

## 4. 版本阶段

### V3.0 桌面基础层

V3.0 的目标是让当前系统具备桌面运行的基础条件，但还不追求正式安装包体验。

重点包括：

- 增加 `server` 和 `desktop` 两种运行模式。
- 桌面模式使用用户目录保存数据、备份、日志和设置。
- Windows 默认数据根目录为 `%LOCALAPPDATA%\SecretBase\`。
- 桌面模式后端只监听 `127.0.0.1`，端口由启动器动态分配。
- 前端 API 地址由启动器注入，不再依赖固定 `127.0.0.1:10004`。
- 保持服务器部署模式的默认行为不变。

V3.0 的成功标准是：在开发环境中可以用桌面模式启动本地后端和前端，并且不读写仓库内的真实运行数据目录。

V3.0 的实现说明见 [SecretBase V3.0 桌面基础模式](v3-desktop-foundation.md)。

状态：已在 `v3.0.0` 完成并进入持续回归。

### V3.1 Windows 桌面 MVP

V3.1 的目标是交付第一个可用的 Windows 离线桌面版本。

重点包括：

- 使用 PyInstaller 打包 Python 后端、静态前端和 pywebview 桌面壳。
- 启动桌面应用时自动拉起本地后端。
- 桌面窗口打开后可以完成初始化、解锁、条目 CRUD、搜索、标签、回收站、备份中心、导入导出和恢复。
- 关闭桌面窗口时停止本地后端进程。
- 应用包内不得包含真实 vault、日志、`.env`、备份文件或用户数据。

V3.1 不包含自动更新、代码签名、系统托盘、开机启动、浏览器自动填充和云同步。

V3.1 的实现和验收状态见 [SecretBase V3.1 Windows 桌面 MVP 实现与验收](v3.1-windows-desktop-mvp.md)。

状态：已通过 Windows 10/11 x64 真机验收并在 `v3.1.0` 正式发布。

### V3.2 桌面产品化增强

V3.2 的目标是让桌面应用更接近日常可长期使用的状态。

已锁定的实现范围：

- 当前用户免管理员 Windows 安装器，同时保留便携 ZIP。
- 数据、备份和日志目录入口，以及可复制的脱敏桌面诊断。
- 手动检查 GitHub 正式版本，不做应用内自动更新。
- 自由调整窗口尺寸，并在 WebView2 原生缩放时显示当前比例。
- 可选系统托盘，关闭到托盘前强制锁定 vault。
- 默认卸载保留数据；双确认后才允许删除整个本地数据目录。
- 预留代码签名接口，但签名证书不作为开发构建前置条件。

本阶段仍不记住主密码，不把主密码保存到系统钥匙串。

状态：已完成。V3.2 已通过 Windows Server 2022/2025 CI 和 Windows 10/11 x64 真机验收，并在 `v3.2.0` 正式发布。

### V3.3 macOS 桌面版本

V3.3 在 Windows 桌面链路稳定后推进，继续复用同一套 Python 后端、静态前端和桌面数据规则。

重点包括：

- pywebview/PyInstaller 的 macOS 打包适配。
- 应用包资源路径、窗口生命周期和本地端口验证。
- `.app` 产物、图标、签名与公证方案评估。
- `~/Library/Application Support/SecretBase` 数据目录兼容。
- 与 Windows 版本通过加密备份互相迁移。

macOS 阶段不先于 Windows MVP，也不单独分叉业务功能实现。

已锁定的首版范围：macOS 13+、Apple Silicon arm64、DMG 与 ZIP、WKWebView、单实例、原生目录与文件选择框。首版不提供菜单栏驻留、签名和公证；Windows 与 macOS 在 `v3.3.0` 使用统一版本并同时发布。

状态：已完成。V3.3 已通过 macOS arm64 CI、下载产物复测和 Apple Silicon 真机验收，并在 `v3.3.0` 正式发布。

### V4.0 共享 vault 核心准备

V4.0 的目标是为手机 App 和未来跨平台能力建立稳定的数据核心边界。

重点包括：

- 固化 vault 文件格式说明。
- 建立跨语言兼容性测试样例。
- 明确加密、解密、备份读取、备份写入、恢复校验的输入输出规则。
- 评估是否将 vault/crypto/storage 核心抽象为 Rust 或其他可跨桌面和手机复用的实现。

已锁定范围为“规范 + 黄金向量 + Rust 原型”：生产 Web 和桌面继续使用 Python；Rust 只提供纯内存格式能力和跨语言验证，不接入 FastAPI、文件锁、备份或桌面包。Vault V1 不迁移，未知 JSON 字段必须保留。

状态：已完成。格式规范、Python payload 边界、公开黄金向量、Rust 参考核心和跨平台发布门禁均已建立，并通过 Linux、Windows、macOS 自动化检查及 Apple Silicon 真机验收。详见 [V4 Vault 核心验收记录](v4-vault-core.md)。

### V5.0 手机 App

V5.0 的目标是实现完全离线的手机 App。UI 采用 Flutter，vault 能力通过 Rust 提供；Android 先行，稳定后适配 iOS。

状态：核心实现完成，尚未发布。Flutter/Rust、私有数据目录、生命周期安全、条目、标签、密码组、回收站、加密迁移和五项需确认的 AI 能力均已落地；三 ABI CI 与 API 29/36 模拟器已经通过，下一门禁是 Android 10+ arm64 真机、跨桌面迁移和持久签名验收。详细边界见 [V5 Android 实施计划](v5-android-plan.md)，真机步骤见 [V5 Android 真机验收清单](manual-qa-checklist-v5-android.md)。

初版手机 App 只关注核心密码库能力：

- 本地创建和解锁 vault。
- 条目浏览、搜索、查看、复制和编辑。
- 标签和回收站基础操作。
- 手动加密备份导入和导出。
- 与桌面版通过加密备份文件进行迁移。
- 由用户预览和确认后才应用的 AI 解析与整理。

手机 App 初版不做账号系统、云同步、多设备实时同步、浏览器自动填充和团队协作。

## 5. 数据和迁移原则

SecretBase 的数据边界继续以加密 vault 和加密备份为核心：

- vault 文件格式保持稳定，不能为了 App 化随意破坏兼容性。
- 手动加密备份是桌面和手机之间的第一迁移媒介。
- 明文 JSON 只作为显式确认后的导出能力，不作为推荐迁移方式。
- 自动备份只负责本机恢复安全，不承担跨设备同步职责。
- 任何格式变更都必须先有备份、恢复和兼容性测试。

## 6. 安全边界

App 化不能降低当前安全模型：

- 主密码不保存明文。
- 不提供“忘记主密码后找回”能力。
- 桌面首版不记住主密码。
- 桌面后端只监听本机地址。
- 桌面模式默认不暴露局域网访问。
- 应用包和安装器不得包含用户数据。
- 日志继续脱敏，避免记录密码、token、API key 和备份主密码。

桌面应用的便利性必须服从本地数据安全边界。

## 7. 明确非目标

以下能力不进入近期 App 化路线：

- 多用户、组织、团队协作和共享 vault。
- 云同步、官方云服务或账号体系。
- 浏览器插件和自动填充。
- 自动更新和发布签名作为桌面 MVP 的必选项。
- 手机端内嵌 Python/FastAPI。
- 使用明文 JSON 作为主要跨设备迁移方式。
- 保存或自动填充主密码。

## 8. 下一步实施重点

V3.2 桌面产品化已经完成安装、升级、卸载数据安全、窗口缩放、关闭确认、托盘锁定、单实例、隐私扫描和双版本 Windows CI 验收。

V5.0 已完成 Flutter Android 工程、Rust FFI、私有存储、生命周期锁定、主要业务功能、三 ABI CI 和 API 29/36 模拟器。本阶段只继续处理真机验收、持久签名密钥、跨桌面备份迁移和发布文档，不创建未经验收的标签。Android MVP 稳定后再适配 iOS，移动端始终不内嵌 FastAPI。
