# SecretBase V5 Android 真机验收清单

本清单用于 Android 10+ arm64 真机和正式签名候选 APK。只使用空 Vault、公开测试向量或专用测试数据，不导入生产 Vault。

状态：2026-07-15 已通过 GitHub 三 ABI 构建、APK 隐私扫描和 API 29/36 模拟器；移动 UI 已完成第二轮优化，下列真机、迁移和持久签名项目仍待验收。

## 安装与启动

- [ ] APK 可在 Android 10（API 29）和较新 Android 版本安装、启动和覆盖升级。
- [ ] 启动图标、应用名、明暗主题和系统返回行为正常。
- [ ] 首次启动只提供创建主密码，不存在默认主密码。
- [ ] 错误主密码无法解锁，正确主密码可进入条目页。
- [ ] 最近任务缩略图和系统截图不会泄露 Vault 内容。

## 生命周期与锁定

- [ ] 切到后台后立即出现内容遮挡，5 分钟后恢复必须重新解锁。
- [ ] 设备锁屏后应用立即锁定，解锁设备返回应用时显示主密码页。
- [ ] 强制结束并重启应用后不会恢复已解锁会话。
- [ ] 修改主密码后旧密码失效，新密码可解锁现有数据和恢复副本。

## 指纹解锁

- [ ] 设置页只有在设备已录入强生物识别时才能开启指纹解锁，开启前必须再次验证当前主密码。
- [ ] SecretBase 只接受指纹/强生物识别，不允许系统 PIN、图案或锁屏密码代替；取消后仍可输入主密码。
- [ ] 开启后强制结束应用或重启设备，进入解锁页会自动弹出一次指纹验证，取消后可通过按钮再次触发。
- [ ] 连续失败或系统锁定指纹后明确回退主密码，不出现空白、卡死或重复弹窗。
- [ ] 增删系统指纹、修改主密码、导入 Vault 或恢复记录后，旧设备凭据失效并要求重新开启。
- [ ] 无生物识别硬件、未录入指纹和 Keystore 暂不可用的设备仍可完整使用主密码流程。
- [ ] 卸载重装后不存在旧指纹凭据；日志、APK 和备份中不包含设备解锁密钥。

## 条目、标签与密码组

- [ ] 条目新建、编辑、详情、模板复制、移入回收站、恢复和彻底删除正常。
- [ ] 自定义字段的可复制和隐藏开关互不干扰，列表不会返回隐藏值明文。
- [ ] 手机卡片最多预览 2 个字段、宽屏最多预览 3 个字段，并正确提示剩余字段数量。
- [ ] 搜索、标签筛选、密码组筛选和分页切换后结果与计数一致。
- [ ] 标签每页 5、10、20、50 条均可滚动访问，批量删除会同步移除条目关系。
- [ ] 密码组可新建、编辑、删除、拖动排序，并可跳转到对应条目筛选结果。
- [ ] 密码组和标签卡片的“编辑/删除”按钮可打开底部操作面板，长名称、大字体、矮屏和 320px 窄屏不溢出。
- [ ] 删除密码组会明确提示仅移除组关系而不删除条目；从密码组或标签进入条目后，返回和清除会回到对应来源页。

## AI 确认流程

- [ ] 未配置 AI 时，所有本地密码库功能仍可正常使用。
- [ ] AI 页默认进入对话式管家；历史、新对话、服务设置和五项专业工具入口在窄屏与横屏均可访问。
- [ ] 输入区只保留紧凑上下文、圆形 `+`、输入框和发送按钮；`+` 面板中的分类、标签、密码组、字段、模式、范围和专业工具在大字体窄屏不溢出。
- [ ] AI 请求处理中仍可打开专业工具和服务设置，但生成、验证或保存第二个冲突请求会被禁用。
- [ ] 内置 OpenAI、DeepSeek、Kimi、智谱 GLM、SiliconFlow、Gemini、OpenRouter 和自定义接口，不出现 Qwen；Base URL 可编辑和恢复默认值。
- [ ] 模型列表失败后可手工填写模型 ID；HTTP Base URL 被拒绝，HTTPS 配置可验证并加密保存。
- [ ] 首次发送前出现隐私同意；普通管家发送清单只包含标题、hostname、分类、字段名和字段属性。
- [ ] 普通管家请求体不包含字段值、完整 URL、备注或真实 UUID；模型计划使用 `E001`、`E001.F01` 临时引用。
- [ ] 普通模式粘贴疑似密码、Token 或私钥会被阻止；只有切换“AI 新建”并二次确认后才能发送完整原文。
- [ ] AI 新建原文不会出现在加密对话历史中，也不会进入后续普通对话上下文。
- [ ] 模型返回 `value`、未知引用、删除条目/字段/密码组或混合标签与密码组计划时被拒绝。
- [ ] 允许的计划逐项勾选后才能应用；用户取消、取消勾选或 Vault revision 变化时不写入。
- [ ] AI 计划按领域分组，详情可独立展开；标签删除与合并默认不选，用户主动勾选后才可应用。
- [ ] 字段拆分只在本机复制原字段值，原字段仍保留；已有条目的 URL 和备注不会被 AI 修改。
- [ ] 写入前产生加密恢复副本；主密码修改后旧 AI 配置和对话历史不能被旧密钥读取。
- [ ] 应用 AI 计划后立即出现“撤回”入口；revision 未变化时可恢复应用前完整状态，并同步刷新条目、标签和密码组。
- [ ] AI 应用后发生任何其他 Vault 写入、锁定或重启时，旧撤回入口失效且不能覆盖后续修改。
- [ ] 文本解析、单条目标签整理、密码组整理、标签治理和操作计划仍可从“专业工具”独立使用。

## 备份、恢复与数据语义

- [ ] 导出使用系统文件创建器，导出的加密备份可被桌面版解锁。
- [ ] 桌面版导出的 Vault 可在 Android 预览数量并确认替换。
- [ ] 错误备份密码、损坏文件和超过 50 MB 的文件均被拒绝，原 Vault 不变。
- [ ] 写入前后产生恢复副本，最多保留 10 份，恢复后条目和未知扩展字段不丢失。
- [ ] 默认应用数据只位于私有目录；卸载会删除该目录，重新安装不会出现旧数据。
- [ ] 卸载前导出的加密备份可在重新安装后恢复。

## 返回与退出

- [ ] 弹窗、底部面板、专业工具和回收站优先正常关闭或返回，不触发退出提示。
- [ ] 从密码组或标签来源进入的筛选条目优先返回对应来源页；其他主标签返回无筛选条目首页。
- [ ] 条目首页和解锁页第一次返回提示“再按一次退出”，2 秒内第二次返回正常退出，超时后重新计时；重新打开应用必须停留在主密码/指纹解锁页。
