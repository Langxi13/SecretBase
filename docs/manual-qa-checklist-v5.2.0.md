# SecretBase V5.2.0 人工验收清单

## 1. 准备

1. 准备一个只用于测试的 HTTPS WebDAV 目录和应用密码，确认服务明确支持强 ETag 与条件写入。
2. 准备两套彼此隔离的 SecretBase 数据目录，可组合使用 Web、Windows 或 macOS；不要使用生产 Vault 做冲突测试。
3. 两端分别导出可用的加密备份，并记录测试条目、标签、密码组和回收站数量。
4. 准备一个包含明显公开测试值 `public-sync-test-value` 的条目，用于确认远端没有明文。
5. Android 仅做统一版本覆盖和原功能回归，不应出现同步入口。

## 2. 覆盖升级与基础回归

1. 从 V5.1.3 覆盖安装 V5.2.0 Windows 和 macOS，确认原 Vault、备份、设置、AI 配置和更新偏好保留。
2. 自托管 Web 部署新版本后强制刷新一次，确认顶部出现紧凑同步状态按钮，登录和原工作台无白屏。
3. 打开条目、密码组、标签、备份、AI 管家、回收站和设置，确认原有写入和导航正常。
4. Android 从 V5.1.3 正式签名包覆盖到 V5.2.0，确认 Vault、指纹、自动填充、AI 和分页偏好保留。

## 3. 创建同步空间

1. 在设备 A 打开“设置 > 同步”，确认默认显示未配置，且同步不会自动联网。
2. 点击“创建同步空间”，填写 WebDAV 地址、用户名、应用密码和设备名。
3. 输入错误密码测试连接，确认只显示 WebDAV 认证失败，应用仍保持解锁。
4. 输入正确配置测试连接，确认通过强 ETag 和条件写入检查。
5. 创建同步空间，确认完成后要求再次输入主密码，不直接展示恢复码。
6. 输入错误主密码，确认受控报错；输入正确主密码后显示二维码和分段恢复码。
7. 关闭窗口后重新打开“加入新设备”，确认必须再次输入主密码，旧恢复码没有保存在输入框或浏览器存储。
8. 在 WebDAV 管理界面下载或搜索远端对象，确认找不到 `public-sync-test-value`、条目标题、字段名、标签或设备名明文。

## 4. 加入第二台设备

1. 在设备 B 的空 Vault 中选择“加入现有空间”，填写相同 WebDAV 凭据和恢复码。
2. 确认加入后条目、回收站、标签、密码组、简介和排序与设备 A 一致。
3. 确认设备 B 可以使用与设备 A 不同的本地主密码解锁和同步。
4. 在有本机测试数据的 Vault 中尝试加入，未勾选合并时应明确阻止；勾选后三方合并。
5. 加入请求使用错误 WebDAV 密码后，关闭并重开同步设置，确认没有残留半配置状态。
6. 制造加入冲突后关闭冲突弹窗，确认设置页只显示“继续处理冲突”和“取消加入”，不会出现历史、轮换或立即同步等尚不可用操作。

## 5. 自动同步触发

1. 在设备 A 新建条目，5 秒内继续编辑一次，确认最终只在防抖结束后同步且界面不卡顿。
2. 在设备 B 点击“立即同步”，确认新条目出现且条目/标签/密码组页面立即刷新。
3. 锁定并重新解锁设备 B，确认自动检查一次且不会重复循环请求。
4. 将桌面窗口隐藏或浏览器切到后台超过 60 秒，在设备 A 修改数据后返回设备 B，确认自动拉取。
5. 关闭“自动同步”，重复写入和前台返回，确认不自动请求；手动同步仍正常。
6. WebDAV 临时断网后写入，确认本机操作成功、同步状态显示失败；恢复网络后手动同步成功。

## 6. 三方合并与冲突

1. 两端基于同一版本分别新建不同条目，依次同步，确认自动合并且两条都存在。
2. 两端修改同一条目的不同标题，后同步的一端必须进入冲突界面，不能静默覆盖。
3. 确认冲突只显示标题、状态、时间、字段数量和变化区域，不显示任何字段值。
4. 分别验证“保留本机”“保留远端”和“保留两份”；保留两份后本机副本标题带“（本机冲突副本）”且 UUID 不重复。
5. 一端删除到回收站，另一端修改同一条目，确认出现删除/修改冲突。
6. 两端修改同一标签、密码组简介和 Vault 排序，确认逐项冲突或安全自动合并。
7. 冲突出现后在本机再修改任意 Vault 数据，旧冲突应用必须提示失效并要求重新同步。
8. 冲突出现后锁定再解锁，旧冲突不得继续应用或泄露旧会话标题。

## 7. 历史、恢复与删除

1. 连续产生至少 12 个同步版本，确认历史界面最多显示 10 项且分页/内部滚动不会隐藏操作。
2. 选择旧历史恢复，确认它成为新的最新版本，两端同步后内容一致。
3. 彻底删除测试条目并产生足够新版本，确认旧快照逐步淘汰。
4. 点击“断开本机”，确认本机 Vault 保留，另一设备和 WebDAV 数据不受影响；使用恢复码可重新加入。
5. 删除远端数据时，错误主密码或未输入 `DELETE` 必须阻止；正确双确认后本机 Vault 保留、同步状态回到未配置。

## 8. 主密码、连接和密钥轮换

1. 同步正常时修改设备 A 主密码，重新打开应用后确认状态、历史和手动同步仍可用。
2. 修改 WebDAV 设备名，用户名和密码留空保存，确认继续使用原凭据。
3. 填入错误的新 WebDAV 密码，保存必须失败且旧配置继续可用。
4. 轮换同步密钥前保存旧恢复码；输入主密码轮换后确认历史只剩一个新版本。
5. 设备 B 使用旧密钥同步必须失败；使用新恢复码重新加入后恢复正常。
6. 导入一个不同 `vault_id` 的加密 Vault，确认旧同步自动停用且不会向旧远端上传新 Vault。

## 9. TLS 与 WebDAV 兼容性

1. 使用 HTTP 公网地址、带账号的 URL、查询参数或片段，确认前端/后端明确拒绝。
2. 使用自签名或过期证书，确认没有“忽略证书”入口，连接失败不锁定应用。
3. 使用返回弱 ETag 或忽略条件头的测试服务，能力测试必须拒绝。
4. 确认能力测试结束后没有遗留随机探测 Vault 目录。

## 10. UI 与生命周期

1. 在亮色、暗色和按时间主题下检查顶部同步按钮、状态色、设置摘要、冲突和历史列表可读。
2. Windows/macOS 将窗口缩到 `360 × 320`，确认同步入口、表单、按钮和恢复码可通过页面滚动访问，文字不重叠。
3. 常规桌面宽度下确认冲突双栏、二维码和连接表单利用宽度合理，不显得过窄。
4. 同步请求进行中仍可查看其他非冲突页面；按钮有处理中状态，不会重复提交或卡死。
5. 恢复码窗口打开时锁定、关闭到托盘或退出，恢复后必须停留在解锁页，二维码、主密码和 WebDAV 密码已清空。
6. 浏览器刷新后确认静态资源使用 V5.2.0 缓存版本，不出现旧 UI 或未定义变量错误。

## 11. Android 回归

1. 确认 Android V5.2.0 没有未完成的 WebDAV 设置或同步入口。
2. 连续十次后台返回和指纹认证，确认无半屏白屏、残留键盘高度或循环认证。
3. 完成条目 CRUD、标签/密码组、回收站、AI 管家应用/撤回和系统自动填充。
4. 检查更新、下载和覆盖安装流程正常，弱网失败仍可重试。
5. 新建测试 Vault 后导出加密备份，确认根文档包含有效 `vault_id` 且不影响桌面导入。

## 12. 发布判定

以下全部满足方可发布：

- 两台真实设备在同一 HTTPS WebDAV 上完成创建、加入、自动同步、冲突、历史、改密、轮换和重加入。
- 远端对象、日志、浏览器存储和发布产物均无恢复码、WebDAV 密码、同步密钥或 Vault 明文。
- 弱 ETag、过期 ETag、错误凭据、TLS 错误和断网均为受控失败，没有静默覆盖、锁死或误锁定。
- Vault V1 旧文件、Windows/macOS 数据目录和 Android 覆盖升级均保持兼容。
- 完整发布门禁、Windows/macOS 打包验证、Android API 29/36 冒烟和正式签名资产检查全部通过。
