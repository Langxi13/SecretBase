# SecretBase V5.1.1 发布评估

## 结论

V5.1.1 是不改变 Vault V1、数据目录、正式签名或业务模型的稳定性候选。范围只覆盖 Android 更新交互、网络诊断、发布权限门禁和 macOS DMG 构建可靠性。完成三端候选包自动化检查和关键人工回归后再创建正式标签。

## 已确认问题

- V5.0.0 至 V5.0.2 的 Android Manifest 缺少 `ACCESS_NETWORK_STATE`。默认开启自动预下载时，客户端在成功发现新版本后读取网络类型会抛出原生权限异常，最终错误地显示“检查更新失败”。V5.1.0 已声明该权限，V5.1.1 进一步把权限纳入源码和成品 APK 双重门禁。
- Android 更新控制器过去把网络状态读取、下载和安装启动异常统一转为不可重试的错误状态。用户需要重新检查更新，取消系统安装后也可能长期停留在处理中。
- V5.1.0 macOS 正式发布首次执行时，应用、ZIP 和自检均成功，但 `hdiutil create` 瞬时失败并留下 `diskimages-help` 孤立进程；重跑同一提交后成功。

## 修复范围

- 发现更新后无法读取网络类型时保留版本与手动下载入口。
- 下载失败或取消后保留下载重试入口；安装界面启动失败或用户取消系统安装后保留安装重试入口。
- 区分超时、DNS/网络、VPN、TLS/代理证书和 HTTP 状态错误。
- 正式 APK 必须声明联网、网络状态和安装包权限。
- DMG 创建、挂载和卸载最多重试三次，逐次等待并保留原始诊断输出。

## 自动化门禁

- 更新服务测试覆盖签名清单、临时签名迁移、清单篡改、断网和 TLS 失败。
- 更新控制器测试覆盖网络状态异常、下载失败、安装启动失败和打开系统安装界面后的可重试状态。
- 发布检查同时读取 Android Manifest 和 APK 验证脚本，防止权限契约只存在于一侧。
- macOS 构建与下载包复测共用同一 `hdiutil` 重试实现。
- 正式候选仍需通过 Python、前端、Flutter、Rust、Windows、macOS、Android API 29/36 和敏感文件扫描门禁。

## 不在本版本解决

- V5.0.x 已安装代码无法被远程修改。用户可关闭“自动预下载”后检查更新，或手动覆盖安装正式 V5.1+ APK。
- 国内网络无法访问 GitHub 时仍需 VPN、可信代理或手动下载；没有可信独立镜像前不引入第三方更新地址。
- Windows Authenticode、macOS Developer ID 签名与公证需要外部证书。
- Android OEM、浏览器和输入法自动填充兼容性必须在真实设备上验证。

## 发布条件

1. 主分支全部 CI 成功，macOS DMG 首次构建成功或重试后留下明确日志。
2. 永久签名 Android 候选可以覆盖 V5.1.0，Vault、指纹凭据和偏好保持不变。
3. Windows 候选覆盖安装后数据、托盘和缩放保持不变。
4. macOS 候选手动覆盖后应用支持目录保持不变。
5. 完成 `docs/manual-qa-checklist-v5.1.1.md` 中的发布阻断项。
