# SecretBase V5.1.1 逐步人工验收清单

本清单只使用专用测试 Vault 或完成加密备份后的个人 Vault。任何卸载、清除数据、临时签名迁移和错误包测试都必须先确认备份可读取。

## 1. 准备与记录

- [ ] 在现有版本中创建一份加密备份，并把文件复制到应用数据目录以外的位置。
- [ ] 使用正确主密码预览该备份，确认条目、标签、密码组和回收站数量合理。
- [ ] 记录升级前版本、平台、系统版本、设备型号、Vault 条目数、标签数、密码组数和指纹解锁状态。
- [ ] 准备至少三个测试条目：普通账号、含隐藏字段条目、含超长邮箱和 URL 的条目。
- [ ] 准备一个测试密码组和一个测试标签，并让同一条目同时属于两者。
- [ ] 正式包只从本仓库 GitHub Release 下载，先核对 `SHA256SUMS.txt`，不要使用来源不明的转存包。

## 2. Web 与通用功能

- [ ] 使用空测试数据目录启动 Web 版，创建主密码后正常进入首页。
- [ ] 错误主密码只提示密码错误，正确主密码可以解锁。
- [ ] 新建、详情、编辑、“在此基础上新建”、删除、恢复和彻底删除条目均正常。
- [ ] 自定义字段“可复制”和“隐藏”互不影响；隐藏字段在列表和 AI 详情中不显示明文。
- [ ] 超长邮箱和 URL 在卡片中单行省略，详情和编辑中仍能查看完整内容。
- [ ] 标签和密码组支持新增、编辑、删除、分页和条目关系维护；删除密码组不删除条目。
- [ ] 从密码组详情新建条目时默认带入当前密码组；从标签来源新建时默认带入当前标签。
- [ ] 从密码组或标签进入条目后，清除筛选和返回会回到正确来源页。
- [ ] AI 未配置时所有本地功能正常；配置测试服务后，发送前必须显示实际数据范围确认。
- [ ] AI 普通管理请求不包含字段值、完整 URL、备注或真实 UUID；混合计划可以分组审核、一次应用和一次撤回。
- [ ] 锁定、刷新页面和自动锁定后不会残留旧条目画面或保护遮罩。

## 3. Windows V5.1.1

- [ ] 在 Windows 10 或 Windows 11 x64 上记录 V5.1.0 的数据目录和当前缩放比例。
- [ ] 退出应用和托盘进程，运行 V5.1.1 安装器执行覆盖安装，不选择删除本地数据。
- [ ] 安装完成只启动一个实例，不出现额外控制台窗口。
- [ ] 使用原主密码解锁，确认 Vault、备份、设置、AI 配置、托盘偏好和缩放比例均保留。
- [ ] 将窗口缩到约 `360 x 320`，登录、条目列表、设置和弹窗仍可滚动且不重叠。
- [ ] 使用 `Ctrl + +`、`Ctrl + -`、`Ctrl + 0` 和 `Ctrl + 滚轮`，比例提示和布局变化正常。
- [ ] 关闭窗口分别测试取消、隐藏到托盘和退出；隐藏或托盘锁定后恢复必须立即显示解锁页。
- [ ] 从控制面板卸载时应用和托盘自动退出；默认卸载保留 `%LOCALAPPDATA%\SecretBase`。
- [ ] 重新安装后仍能解锁原 Vault；仅在单独测试数据上验证输入 `DELETE` 的完整数据清除。
- [ ] 设置页检查更新失败时不影响解锁和条目操作，错误信息能区分 404、网络和 TLS 问题。

## 4. macOS V5.1.1

- [ ] 在 macOS 13+ Apple Silicon 上核对 DMG、ZIP 和 `SHA256SUMS.txt` 来自同一正式 Release。
- [ ] DMG 可以挂载并显示 `SecretBase.app` 与 Applications 链接；ZIP 解压得到完整应用。
- [ ] 退出旧版后将 V5.1.1 拖入 Applications 覆盖旧版，不删除应用支持目录。
- [ ] 使用原主密码解锁，确认 Vault、备份、设置、AI 配置和缩放比例均保留。
- [ ] `Command + +`、`Command + -`、`Command + 0` 正常，普通文本输入不会误触发缩放。
- [ ] 关闭、取消关闭、`Command + Q`、重复启动和异常启动均无卡死或残留后台进程。
- [ ] 原生导入、加密导出和明文导出文件选择器正常；外部网址由默认浏览器打开。
- [ ] 未签名提示只能通过“隐私与安全性 -> 仍要打开”处理，不关闭 Gatekeeper。
- [ ] 手动检查更新能显示正式版本或“当前已是最新版本”，断网失败不影响本地功能。

## 5. Android 正式签名升级

- [ ] 确认当前安装的是 GitHub Release 的正式签名 V5.1.0，而不是文件名带 `-ci` 的测试包。
- [ ] 在 V5.1.0 中确认主密码、指纹解锁、分页数量、AI 配置、标签和密码组均正常。
- [ ] 在 V5.1.0 设置页点击检查更新，下载永久签名 V5.1.1 并进行覆盖升级，不卸载旧版。
- [ ] 系统允许覆盖安装且不提示签名冲突；设置页显示 `5.1.1`，`versionCode` 为 `5010100`。
- [ ] 使用原主密码和指纹分别解锁，确认 Vault、设备凭据、分页偏好、AI 配置和自动填充设置保留。
- [ ] 后台切换立即遮挡内容；超过自动锁定时间、锁屏和强制结束后都必须重新解锁。
- [ ] 连续返回第一次提示“再按一次退出”，两秒内第二次正常退出；重新打开必须处于锁定状态。

## 6. Android 更新交互

- [ ] 在 V5.0.0 至 V5.0.2 正式版中关闭“自动预下载”，点击检查更新后能够发现 V5.1.0，并可手动下载或直接使用正式链接升级。
- [ ] V5.1.1 开启自动检查和自动预下载后，当前没有更高正式版本时显示“当前已是最新正式版本”。
- [ ] 断开网络后点击检查更新，提示检查网络、DNS 或 VPN，不出现白屏、卡死或按钮永久禁用。
- [ ] 将系统时间明显调错或使用会替换 HTTPS 证书的测试代理，确认提示检查系统时间、VPN 或代理证书；测试后恢复正确时间和网络。
- [ ] 下载过程中断网后仍保留“下载更新”入口；恢复网络后无需重启应用即可重试。
- [ ] 取消下载后仍保留版本信息和下载按钮。
- [ ] 未授予“安装未知应用”时进入 SecretBase 专属系统授权页，返回后仍可点击安装。
- [ ] 打开 Android 系统安装界面后选择取消，返回应用不会停留在处理中，仍可再次点击安装。
- [ ] 确认安装前 Vault 已锁定；取消系统安装后重新进入应用仍要求主密码或指纹。
- [ ] 临时签名 CI 包检查更新时明确要求导出备份并迁移，不尝试覆盖正式包。

## 7. Android 自动填充

- [ ] 在系统设置中选择 SecretBase 为当前自动填充服务，应用设置页只在确实选中时显示已启用。
- [ ] Chrome 登录页和至少一个原生应用登录页会显示建议；选择候选前必须先指纹或主密码验证。
- [ ] 候选阶段只显示标题、账号预览和字段名；取消验证或超过 120 秒后不能填充。
- [ ] 完整域名匹配优先于同站点匹配；手动字段映射不能把同一字段同时用作账号和密码。
- [ ] Android 11+ 输入法行内建议开关生效，关闭后系统下拉建议仍可使用。
- [ ] 注册或修改密码页面不会填入已有密码；提交新登录信息后可确认保存为新条目。
- [ ] 分步登录先账号后密码时保存提示仍包含账号；不同网站或应用不会错误合并。
- [ ] “不再为此网站或应用显示”立即生效，设置页清除停用列表后恢复建议。
- [ ] 自动填充保存新条目后，已打开的主应用列表和分类计数及时刷新。

## 8. Android 数据与 AI

- [ ] Android 导出的加密备份可在 Windows 或 macOS 预览并导入，条目、隐藏字段、标签和密码组一致。
- [ ] 桌面导出的加密备份可在 Android 预览并替换；错误密码和损坏文件不会改变原 Vault。
- [ ] 导入、恢复或修改主密码后旧指纹凭据失效，需要重新开启。
- [ ] AI 普通模式发送清单不包含字段值；疑似密码、Token 或私钥提示词会被阻止。
- [ ] AI 新建模式必须二次确认；敏感原文不进入历史或后续普通对话上下文。
- [ ] AI 应用计划后出现撤回入口；任何后续 Vault 写入、锁定或重启都会让旧撤回失效。

## 9. 发布阻断判定

出现以下任一情况不得发布：Vault 无法解锁、数据或分类丢失、隐藏值泄露、正式签名不能覆盖、更新后指纹凭据异常、取消安装后永久卡死、自动填充未经验证即返回密码、AI 发送已有字段值、安装包包含用户数据或三端校验和不一致。

允许记录后发布的非阻断项：国内网络无法直连 GitHub、Windows 未配置 Authenticode、macOS 未签名公证、特定第三方应用不支持标准 Android 自动填充但 Chrome 和基准原生应用正常。
