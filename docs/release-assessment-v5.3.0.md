# SecretBase V5.3.0 发布评估

## 结论

V5.3.0 可以进入正式发布流程。该版本把 Web、Windows、macOS 和 Android 的用户自有 WebDAV 同步统一到 V2 加密快照协议，默认不依赖强 ETag，适合坚果云等基础 WebDAV 服务。

## 功能范围

- `SBS2` AES-GCM 加密快照，独立 256 位同步密钥，远端不接收字段名和值明文。
- 不可变快照 DAG、并发分支发现、三方合并和脱敏冲突摘要。
- 三台及以上设备的分阶段冲突确认，未处理分支不会被静默消费。
- 恢复码/配对 URI、历史查看与恢复、同步密钥轮换。
- 显式“压缩并清理历史”：要求本机和远端基线一致，切换到新空间根快照；其他设备必须重新加入。
- Web/桌面端和 Android 端均支持 `PROPFIND` 命名空间解析、无 ETag 上传后校验、有限重试和远端删除竞态保护。
- V1 严格 ETag 协议保留，显式迁移到 V2，不自动删除旧 V1 空间。

## 安全边界

- 生产 WebDAV 地址必须为 HTTPS；不提供忽略证书错误的选项。
- WebDAV 用户名和应用密码只在本机加密配置中保存，不写日志、不进恢复码、不上传 AI。
- 恢复码和配对 URI 等同于同步密钥，只有再次验证主密码后显示。
- 冲突摘要不包含字段名、字段值、完整 URL、备注或真实条目 ID。
- 历史压缩不是无感后台清理，必须输入 `COMPACT` 并确认其他设备均已同步。
- 弱 WebDAV 不提供目录树级原子 CAS；客户端会保护未知对象、删除后重新枚举并在变化时停止，但最后一次空目录检查与目录删除之间仍存在服务端竞态。本机 Vault 和压缩后的新空间不会随旧空间清理失败而删除。
- 单快照密文/解压内容限制为 64 MiB，远端历史限制为 1000 个快照、累计 256 MiB 密文和 256 MiB 解密内容、最多 32 个 frontier。
- 本机 Vault、AI 配置、AI 对话历史、自动填充绑定、生物识别凭据和分页偏好不参与同步。

## 已知限制

- 历史压缩后离线设备不会自动迁移，必须用新恢复码重新加入。
- 不提供官方云、账号找回、实时推送或团队共享。
- iOS 继续搁置。
- 真实 WebDAV 服务可能限制请求频率；客户端只做有限重试，不绕过 TLS 或权限错误。

## 发布前人工确认

按 [V5.3.0 人工验收清单](manual-qa-checklist-v5.3.0.md) 完成 Windows、macOS、Android、Web 和双设备/三设备真机验收后，再创建 `v5.3.0` tag。真实 WebDAV 协议验证已完成，但正式发布包仍必须通过隐私扫描、SHA-256 校验和签名检查。
