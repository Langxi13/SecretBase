# SecretBase V3.2 Windows 真机验收清单

本清单只使用空数据目录或专用测试数据，不读取生产 vault。全部阻断项通过后，才允许合并并创建 `v3.2.0` 标签。

## 环境与产物

- [ ] Windows 10 或 Windows 11 x64。
- [ ] 已安装 Edge WebView2 Runtime。
- [ ] 从 V3.2 工作流下载安装器、便携 ZIP 和 `SHA256SUMS.txt`。
- [ ] 两个产物的 SHA-256 均与校验文件一致。
- [ ] 浏览器直接下载后不解除锁定，安装器和 ZIP 均可正常使用。
- [ ] 未配置签名证书时，已确认并记录 SmartScreen 提示属于未签名版本预期行为。

## 安装与兼容

- [ ] 安装器默认不请求管理员权限。
- [ ] 默认安装到当前用户目录，并创建开始菜单入口。
- [ ] 桌面快捷方式选项正常。
- [ ] 安装完成后可以直接启动，没有额外控制台窗口。
- [ ] 已存在 V3.1 数据时，安装版可以直接看到同一 vault，不要求迁移。
- [ ] 覆盖安装后 vault、备份、设置和 AI 本机配置保持不变。
- [ ] 便携版和安装版不会同时启动两个后台实例。

## 桌面状态与更新

- [ ] 初始化页和锁定页都能打开“桌面状态”。
- [ ] 设置页只在桌面模式显示“桌面”页签。
- [ ] 版本、运行方式、Windows、WebView2 和本地服务状态正确。
- [ ] 数据、备份和日志目录均可打开。
- [ ] 复制的诊断信息不包含 Windows 用户名、真实主目录、条目、token 或 API Key。
- [ ] 手动检查更新成功时正确显示最新版本或下载入口。
- [ ] 断网检查更新时只显示错误，不影响条目操作。
- [ ] 启动应用时没有自动发起更新请求。

## 托盘与锁定

- [ ] 默认关闭窗口后直接退出，无残留进程。
- [ ] 启用“关闭窗口时最小化到托盘”后，最小化按钮仍留在任务栏。
- [ ] 点击关闭按钮后窗口隐藏，托盘图标出现。
- [ ] 从托盘恢复后立即显示解锁页，不能先显示旧条目再等交互触发锁定。
- [ ] 托盘“锁定”立即回到锁定页，不能保留锁定前的页面画面。
- [ ] 托盘“退出”后窗口、本地后端和托盘图标全部消失。
- [ ] 隐藏到托盘时再次启动应用，只恢复已有窗口。

## 卸载数据安全

- [ ] 默认卸载后程序目录已删除，但 `%LOCALAPPDATA%\SecretBase` 完整保留。
- [ ] 启用关闭到托盘并保持应用运行时，从控制面板卸载；应用应自动退出，卸载器不得重新打开窗口或要求任务管理器终止进程。
- [ ] 重新安装后可以继续解锁原 vault。
- [ ] 勾选删除本地数据但未输入 `DELETE` 时，卸载器阻止继续清除。
- [ ] 取消删除勾选后可以安全卸载并保留数据。
- [ ] 勾选并输入 `DELETE` 后，卸载器删除整个 `%LOCALAPPDATA%\SecretBase`。
- [ ] 删除失败时明确显示残留目录，不静默声称清理成功。

## 核心功能回归

- [ ] 初始化、解锁、自动锁定和修改主密码正常。
- [ ] 条目增删改查、搜索、标签、密码组、分页和回收站正常。
- [ ] 原生“另存为”导出、导入和备份恢复正常。
- [ ] AI 未配置时核心功能正常；已配置时安全设置仍可读取。
- [ ] 关闭和卸载过程中没有读取、覆盖或打包生产数据。
