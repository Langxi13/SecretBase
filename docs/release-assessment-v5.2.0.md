# SecretBase V5.2.0 发布评估

## 结论

V5.2.0 引入用户自有 WebDAV 的端到端加密同步，首版覆盖自托管 Web、Windows 和 macOS。Android 随统一版本发布但不开放同步 UI；移动 Rust 核心已经验证相同 Sync Bundle V1，为 V5.3 复用协议。

同步默认关闭，不改变 Vault V1 envelope、既有数据目录、备份、AI 隐私边界、自动填充绑定、生物识别凭据或三端更新签名。完成本文自动化门禁和 `manual-qa-checklist-v5.2.0.md` 的双设备真实 WebDAV 验收后，可以发布。

## 实现范围

- 独立随机 256 位同步密钥，不复用主密码或 WebDAV 密码。
- `SBS1` Sync Bundle V1：紧凑 JSON、Gzip、AES-256-GCM 和对象上下文 AAD。
- 加密 head、不可变加密快照、强 ETag 和 `If-Match` / `If-None-Match` 条件提交。
- 最近 10 个远端加密历史版本，以及“恢复为新的最新版本”。
- 基线、本机、远端三方合并；同实体冲突支持本机、远端和条目保留两份。
- 二维码、分段恢复码、主密码二次验证、同步密钥轮换、本机断开和双确认远端删除。
- 解锁、Vault 写入后 5 秒、后台超过 60 秒返回和手动同步四类触发。
- 顶部紧凑状态入口和“设置 > 同步”；配置、冲突、历史与配对使用独立宽版响应式弹窗。
- `sync-settings.enc` 和 `sync-base.enc` 使用 Vault 用途隔离密钥加密；修改主密码时重加密，导入不同 Vault 时清除。

## 安全评估

- WebDAV 上不出现条目、字段、标签、密码组和设备名称明文；服务端仍可观察目录 UUID、密文大小、对象数量和访问时间。
- 恢复码、配对 URI 和二维码等同于同步密钥，创建接口不直接返回，必须再次验证主密码后获取。
- 生产接口只允许 HTTPS；本机 HTTP 仅存在于自动化测试注入的传输适配器中。不跟随重定向，不允许忽略证书错误。
- WebDAV `401/403` 映射为 502 上游错误，不能触发 SecretBase 的 `secretbase:unauthorized` 锁定路径。
- WebDAV GET 使用 64 MiB 流式硬限制；强 ETag 必须符合标准格式，远端删除使用当前 head ETag 条件保护。
- 冲突公开 DTO 不包含字段对象或值；旧冲突绑定解锁会话、Vault revision 和远端 ETag。
- 本机先行替换后远端提交失败时，使用 revision 条件回滚；不会覆盖用户随后产生的新写入。
- 同步密钥轮换会重建当前快照；旧历史延迟到本机新配置落盘后清理，本机保存失败时恢复旧远端密钥和原历史链。

## 兼容性评估

- Vault envelope 保持 `SB01` V1。
- `vault_id` 是可选 Vault V1 根字段。旧文档缺省时继续缺省，不写入 `vault_id: null`；首次启用同步时生成稳定 UUID。
- Windows/macOS 继续使用原数据目录，升级不移动 Vault。
- Android 新建 Vault 开始生成 `vault_id`，但 V5.2 不执行网络同步；旧 Android Vault 仍可正常打开。
- AI 配置/历史、自动填充绑定、生物识别和界面偏好不进入同步快照。

## 已知限制

- V5.2 不提供 Android 同步界面、后台同步任务或扫码加入；计划在 V5.3 复用协议。
- 不支持官方云、账号找回、共享 Vault、团队权限或实时推送。
- 不兼容弱 ETag、忽略条件头或无法稳定返回 ETag 的 WebDAV 服务。
- 历史固定为最近 10 版；彻底删除的数据在相关旧加密快照淘汰前仍可恢复。
- 首版按完整条目合并，不做同一条目内字段级自动合并。
- WebDAV 可观察密文流量元数据，端到端加密不提供匿名网络层。

## 发布阻断

以下任一情况阻断发布：

- WebDAV 上出现任何测试字段明文，或恢复码/密码进入日志与浏览器存储。
- 无条件覆盖 head、过期 ETag 未被拒绝，或同条目双端修改被静默覆盖。
- 错误 WebDAV 密码导致应用退出解锁态。
- 修改主密码、锁定、导入不同 Vault 或密钥轮换后出现配置与远端永久失配。
- 冲突、历史或设置弹窗在窄窗口换行遮挡、无法关闭、卡死或字段值可见。
- V5.1.3 的 Android 生物识别、自动填充、后台锁定、更新或半屏白屏回归失败。
