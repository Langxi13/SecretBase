# SecretBase V5.1.0 发布评估

## 结论

V5.1.0 是 V5 Android 首个系统集成功能版本，核心新增 Android 10+ 原生自动填充，并统一 Web、桌面和 Android 的密码组新建上下文。版本不修改 Vault V1、默认数据目录、发布签名、更新协议或 AI 隐私边界。

## 功能范围

- Android 设置页可启用和管理 SecretBase 系统自动填充服务。
- 指纹或主密码验证后才展示候选，用户明确选择条目后才读取并返回密码。
- 支持网站匹配、已记住绑定、账号/密码字段映射、Android 11+ 行内建议和目标停用列表。
- 注册与修改密码页面不填入已有密码；登录提交后可经二次验证保存为新条目。
- 分步登录保存会在同一应用和域名的最多 8 个系统上下文中合并账号，避免只读取最后页面导致账号丢失。
- Web、桌面和 Android 从密码组或标签来源新建条目时默认继承当前分类，不受当前列表是否为空影响。

## 安全边界

- 自动填充不调用 AI、HTTP 或任何外部网络接口。
- 候选响应只包含条目标题、账号预览、字段名和匹配信息，不包含密码值。
- 密码只在用户最终选择后从 120 秒本机会话中读取；会话最多保留 3 个，取消或超时后失效。
- 字段值不进入 Intent、日志、通知或持久偏好；新登录信息仅在进程内短暂保存。
- 网站绑定和字段映射写入用途隔离的 `autofill-settings.vault`，修改主密码时重新加密，创建或导入 Vault 时清除。
- 保存操作继续使用 Rust Vault 事务、原子替换和恢复副本；自动填充不能删除条目、字段、字段值或密码组。

## 自动化门禁

- Flutter 格式检查和 `flutter analyze` 通过。
- 37 项 Dart/Widget 测试通过。
- Mobile Rust 格式、Clippy `-D warnings` 和 30 项测试通过。
- Android 原生单元测试通过；Release Lint 为 0 error、6 个非阻塞模板或工具链警告。
- 前端密码组上下文运行时测试、模块装配测试和 JavaScript 语法检查通过。
- APK 检查覆盖 Autofill Service、认证 Activity、三 ABI Rust JNI 导出、R8 保留和真实构建路径扫描。
- GitHub 主分支 Android workflow 必须完成三 ABI构建及 API 29/36 模拟器服务注册；正式标签 workflow 必须使用受保护的永久签名。

## 发布后真机回归

- Chrome、一个原生登录页和一个新密码页的填充与保存流程。
- 指纹取消、失败、锁定和主密码回退。
- 已记住网站绑定、手动字段映射、停用目标和行内建议开关。
- 从 V5.0.2 正式 APK 覆盖升级到 `versionCode=5010000`，确认 Vault、指纹凭据和本机偏好保留。
- 自动填充保存后回到主应用，条目和分类立即刷新。

## 后续方向

V5.1 后续优先处理 Android 厂商兼容性、自动填充真机反馈、跨端加密迁移和更新稳定性。iOS 因签名和分发限制继续搁置，不作为下一版本前置目标。
