# SecretBase V5.1.3 人工验收清单

## 1. 准备

1. 在 Android V5.1.2 正式版导出一份加密备份，并确认主密码可用。
2. 开启指纹解锁，记录当前条目、标签、密码组、分页偏好、AI 配置和自动填充状态。
3. 确认设备 CPU 架构；常见近年手机优先使用 `arm64-v8a` 包。
4. 测试期间使用覆盖安装，不要卸载应用；卸载会删除 Android 应用私有数据。

## 2. 版本与覆盖安装

1. 从 V5.1.2 打开“设置 > 版本更新”，确认能发现 V5.1.3。
2. 完成系统覆盖安装，确认包名和永久签名未变化。
3. 打开应用并通过主密码或指纹解锁。
4. 核对 Vault、偏好、指纹、AI 配置和自动填充设置均保留。
5. 在“设置 > 版本更新”确认当前版本为 V5.1.3。

## 3. 半屏白屏重点回归

1. 在全部条目页按 Home 键切到桌面，再从最近任务返回并完成指纹认证。
2. 连续执行十次“后台 > 返回 > 指纹解锁”，每次确认正文填满底部导航上方区域，没有半屏白屏。
3. 分别从条目详情、条目编辑、密码组、标签、AI 管家和设置页各执行两次后台返回。
4. 在条目编辑页先点开软键盘，再按 Home；返回并认证后确认键盘已关闭且布局恢复完整。
5. 在主密码解锁页主动点输入框打开键盘，再切到后台；返回并使用指纹，确认没有残留键盘高度。
6. 快速连续切换最近任务五次，确认不会出现正文压缩、底部导航漂移、空白区域或无法点击。
7. 旋转屏幕或改变系统字体大小后重复上述操作，确认布局仍完整。

## 4. 认证边界

1. 指纹弹窗出现时不要切换应用，确认短暂 `inactive` 不会触发第二个指纹弹窗或循环锁定。
2. 取消自动指纹弹窗，确认停留在主密码解锁页且软键盘不会自动弹出。
3. 点击主密码输入框，确认键盘正常显示、页面可以滚动且提交后完整进入密码库。
4. 关闭指纹功能后从后台返回，确认要求主密码且布局完整。
5. 首次创建新测试 Vault 时，确认主密码输入框仍自动聚焦。
6. 关闭屏幕并重新解锁设备，返回 SecretBase 后确认重新认证且无白屏。

## 5. 核心功能回归

1. 新建、编辑、复制模板、收藏、删除和恢复条目。
2. 在条目编辑、搜索、AI 输入和导入密码输入页检查软键盘弹出与收起均正常。
3. 新建、编辑、删除标签与密码组，并检查分页和来源返回逻辑。
4. 完成一次系统自动填充，确认认证和候选选择正常。
5. 打开 AI 管家，使用无敏感值测试数据完成一次建议、应用和撤回。
6. 连续返回两次退出应用，重新打开后确认需要认证且首屏完整。

## 6. 发布判定

以下全部满足方可确认：

- 多个页面连续十次后台指纹认证均没有半屏白屏、残留键盘高度或卡死。
- 指纹取消、主密码回退、首次创建和系统锁屏路径均正常。
- 普通输入页面的软键盘仍能正确调整布局。
- 正式签名覆盖更新不丢失 Vault、偏好、自动填充或指纹配置。
- Android API 29/36 冒烟、完整 Flutter 测试和 Release 构建全部通过。
