# SecretBase V3.3 macOS arm64 桌面版

本文档记录 V3.3 的实现边界和发布门禁。V3.3 复用现有 FastAPI、静态 Vue 前端、vault 格式和桌面桥，不为 macOS 分叉业务功能。

状态：功能分支实现阶段。代码和自动化完成后，仍需 Apple Silicon 真机完成 DMG、ZIP、Gatekeeper、窗口生命周期和数据迁移验收，才能创建正式 `v3.3.0` 标签。

## 1. 支持范围

- 最低系统版本为 macOS 13，首版只构建 Apple Silicon arm64。
- 数据保存在 `~/Library/Application Support/SecretBase`，目录权限收紧为当前用户可访问。
- 使用 PyInstaller `.app` 和 pywebview Cocoa/WKWebView，应用内启动仅监听回环地址的 FastAPI。
- Windows 与 macOS 共用 `3.3.0` 版本号、前端、API、vault 和加密备份格式。

正式版本同时提供：

```text
SecretBase-v3.3.0-windows-x64-setup.exe
SecretBase-v3.3.0-windows-x64.zip
SecretBase-v3.3.0-macos-arm64.dmg
SecretBase-v3.3.0-macos-arm64.zip
SHA256SUMS.txt
```

## 2. macOS 桌面行为

- DMG 提供拖入 Applications 的安装方式，ZIP 作为备用和排障产物。
- 使用文件锁和当前用户专属 Unix Socket 实现单实例、重复启动聚焦和退出通知。
- 桌面状态显示 macOS、arm64、WKWebView、数据目录和平台能力，诊断摘要继续隐藏真实主目录。
- 数据、备份和日志目录通过系统 `open` 打开；导入导出使用原生文件选择框。
- 关闭窗口时只提供退出和取消；退出前锁定 vault 并停止本地服务。
- 首版不提供菜单栏驻留、关闭到托盘或 WebView2 缩放比例提示，前端根据能力字段隐藏对应入口。

## 3. 未签名测试版

当前阶段没有 Apple Developer Program 账号，不执行 Developer ID 签名和公证。浏览器下载后 macOS 可能阻止首次打开：

1. 将 SecretBase 拖入 Applications。
2. 尝试打开一次应用。
3. 打开“系统设置 -> 隐私与安全性”。
4. 在安全提示中选择“仍要打开”，再次确认启动。

普通用户文档不推荐关闭 Gatekeeper，也不把 `xattr` 作为默认开启方式。正式签名和公证在后续独立版本处理。

## 4. 构建与 CI

Apple Silicon macOS 15 Runner 使用 arm64 Python 3.11：

```bash
scripts/build-desktop-macos.sh
```

构建脚本生成 `.icns`、`.app`、DMG、ZIP 和 SHA-256，并执行后端资源自检、WKWebView 运行时自检、架构检查和敏感文件扫描。

GitHub Actions 使用 `macos-15` arm64 Runner。PR 和主分支会重新下载构建产物，复测 ZIP、DMG、应用元数据、arm64 可执行文件和打包运行时。正式 Release 只有在 Linux、Windows、macOS 发布检查及两个桌面平台构建全部通过后才创建。

## 5. 发布门禁

- 完成 [V3.3 macOS 真机验收清单](manual-qa-checklist-v3.3.md)。
- 确认 Windows V3.3 保持 V3.2 安装、托盘、缩放和卸载行为。
- 确认 macOS 与 Windows 可以通过加密备份双向迁移。
- 确认四个发布产物均不包含 vault、备份、日志、`.env`、设置、绝对路径或凭据。
- 正式标签必须与 `backend/version.py` 的 `3.3.0` 一致。

## 6. 非目标

V3.3 不包含 Intel Mac、Universal 2、Apple 签名与公证、菜单栏驻留、自动更新、开机启动、云同步或移动端应用。
