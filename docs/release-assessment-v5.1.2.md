# SecretBase V5.1.2 发布评估

## 结论

V5.1.2 聚焦 Android 弱网更新、前后台认证边界、指纹凭据误失效和安装包体积。Vault V1、数据目录、正式签名、AI 隐私边界和桌面业务功能保持不变。

候选版本在正式发布前必须通过 GitHub Release 流水线、Android API 29/36 冒烟测试及 `docs/manual-qa-checklist-v5.1.2.md` 的真机关键项。

## 已解决风险

- 更新清单请求执行三次有限重试，临时失败不再被记录为 24 小时内的成功检查。
- APK 下载支持严格校验的 HTTP Range 续传；断网、超时和暂停不会删除有效部分文件。
- 服务器忽略 Range 时从头覆盖，403、416、非法范围、超长响应和哈希错误不会拼接为可安装文件。
- 安装前继续校验大小、SHA-256、包名、版本号和永久签名。
- 应用进入后台后返回即锁库；仅短暂失焦的系统指纹弹窗不会触发循环锁定。
- Android Keystore 临时异常不再清除指纹凭据。永久密钥失效、密文损坏和 Vault 绑定变化仍要求重新开启。
- 发布同时提供通用 APK 与三个 ABI 独立 APK，客户端优先选择设备匹配包并安全回退通用包。

## 体积结果

本机构建采用单 worker，并通过 APK Manifest、Flutter AOT、Rust Vault JNI 和 ABI 校验：

| APK | 实测大小 |
| --- | ---: |
| `armeabi-v7a` | 21.1 MB |
| `arm64-v8a` | 24.1 MB |
| `x86_64` | 25.8 MB |
| V5.1.1 通用包参考 | 65.7 MB |

V5.1.1 及更早客户端只认识 `android-universal`，因此首次升级到 V5.1.2 仍下载通用包，且该次下载仍使用旧版下载器。V5.1.2 安装完成后，更新到后续更高版本时才会启用断点续传并自动选择 ABI 独立包。

## 自动化证据

- `flutter analyze --no-pub`：通过。
- 更新下载、清单选择、控制器和生命周期定向测试：通过。
- Kotlin Release 编译：通过。
- 更新清单生成及桌面更新回归：通过。
- 三个 ABI 独立 APK 构建和 `verify_android_apk.sh`：通过。
- 完整 Flutter、Rust、后端、前端和跨平台发布门禁：由正式 CI 复核。

## 保留行为

- 增删系统指纹、关闭锁屏凭据、修改主密码、导入或恢复其他 Vault 后，旧指纹凭据可能或必须失效，这是安全设计。
- Android 卸载仍会删除应用私有数据；更新安装和同签名覆盖不会删除。
- 指纹不可用或用户取消时必须能回退主密码，不绕过 Vault 认证。

## 发布阻断

以下任一情况阻断发布：

- 断网重试后部分文件被删除，或续传产生哈希不一致但仍进入安装。
- 从后台返回仍显示原敏感页面且无需认证。
- 系统指纹弹窗触发重复弹窗、循环锁定或界面卡死。
- 临时 KeyStore 异常清除了仍可恢复的指纹凭据。
- ABI 独立包缺少对应 Flutter、AOT 或 Rust 原生库。
- 同签名覆盖安装导致 Vault、偏好、自动填充设置或加密历史丢失。
