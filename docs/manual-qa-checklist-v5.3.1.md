# SecretBase V5.3.1 人工验收清单

## 1. Web/桌面创建与配对

- [ ] 在 Web 或 Windows/macOS 桌面端创建 V2 同步空间，输入错误主密码时不能显示恢复码或二维码。
- [ ] 验证后显示二维码、恢复码和配对链接；配对链接中不出现 WebDAV 应用密码或 `key=` 参数。
- [ ] 在另一台 Web/桌面端选择“使用恢复码/配对链接加入”，粘贴配对链接，确认地址、用户名、协议和恢复码自动填充。
- [ ] 粘贴 V5.3.0 旧 V1/V2 配对链接，确认本地转换并识别为对应模式；粘贴无效协议、HTTP 地址、校验错误或 URL 内嵌凭据时给出明确错误。

## 2. Android 扫码加入

- [ ] Android 空 Vault 进入“设置 → WebDAV 加密快照同步 → 加入空间”，点击“扫描二维码”。
- [ ] 首次使用时允许摄像头权限，扫描电脑端二维码后确认地址、用户名和恢复码自动填充。
- [ ] 确认 WebDAV 应用密码没有被二维码填充，必须手动输入后才能加入。
- [ ] 拒绝摄像头权限、扫描普通网址或旧格式无效二维码，确认页面可操作且不会发起网络请求。
- [ ] 点击“粘贴配对链接”或扫描器内的手动粘贴入口，确认与扫码结果一致。
- [ ] 从剪贴板粘贴成功后确认匹配的配对链接被清空；剪贴板内容变化时不误清理新内容。
- [ ] 点击“测试连接”，确认只做 WebDAV 能力探测，不创建快照；错误密码和断网时可重试。
- [ ] 加入成功后关闭同步弹窗再重新打开，恢复码和配对链接不会残留在表单中。
- [ ] 加入过程中锁定应用，确认弹窗关闭或敏感输入清空；重新解锁后可重新开始加入。

## 3. 复制与安全清理

- [ ] Android 显示恢复码后点击“复制恢复码”，确认按剪贴板设置自动清理。
- [ ] 点击“复制配对链接”，在 Web 加入页粘贴并成功读取；链接不包含 WebDAV 应用密码。
- [ ] 锁定、退出或系统切后台后，恢复码、配对链接和 WebDAV 密码不再显示。

## 4. 回归

- [ ] 原有 V5.3.0 Android 正式签名 APK 可覆盖安装 V5.3.1，Vault、设置、同步配置和生物识别状态保持。
- [ ] 已有 V2 同步空间的设备仍可正常手动/自动同步，旧快照不需要迁移。
- [ ] V1 严格 ETag 空间的 Web/桌面加入和同步行为不变。
- [ ] Android 不支持摄像头的设备仍可使用恢复码手动加入。
