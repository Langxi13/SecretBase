# SecretBase V5.1.3 发布评估

## 结论

V5.1.3 是 Android 界面稳定性修复版本，解决连续后台切换和指纹认证后主内容区可能只显示上半屏的问题。Vault V1、数据目录、永久签名、系统自动填充、AI 隐私边界和桌面业务功能保持不变。

候选版本在正式发布前必须通过 GitHub Release 流水线、Android API 29/36 冒烟测试及 `docs/manual-qa-checklist-v5.1.3.md` 的真机关键项。

## 根因与修复

- 锁定页此前会自动聚焦主密码输入框，同时自动发起系统指纹弹窗。部分 Android 或 OEM 系统在重复切换时会留下非零 IME Insets。
- Flutter `Scaffold` 因残留的键盘高度压缩正文，但底部导航仍位于窗口底部，因此中间区域表现为大面积白屏。
- 已锁定状态不再自动聚焦输入框；首次创建 Vault 仍保留自动聚焦。
- 进入后台、设备锁定、指纹认证前后和主密码解锁成功时，主动清除输入焦点并隐藏软键盘。
- Android 生物识别桥在指纹弹窗前后重新请求窗口 Insets 和布局，处理 OEM 窗口状态未及时刷新的情况。
- 修复限定在认证与生命周期边界，没有全局移除 `viewInsets`，条目编辑等正常键盘布局不受影响。

## 自动化证据

- `flutter analyze --no-pub`：通过。
- 完整 Flutter 测试：55 项通过。
- 锁定页焦点、首次创建自动聚焦和后台键盘关闭回归：通过。
- `:app:compileReleaseKotlin` 单 worker Release 编译：通过。
- 完整 Rust、后端、前端、桌面和跨平台发布门禁：由主分支 CI 与正式 Release 流水线复核。

## 保留行为

- 系统指纹弹窗造成的短暂 `inactive` 不视为真正进入后台，避免认证弹窗触发循环锁定。
- 应用进入 `hidden` 或 `paused` 后仍立即保护内容，并在返回时要求指纹或主密码。
- 用户取消指纹后停留在主密码解锁页，只有主动点击输入框才显示软键盘。
- Android 覆盖更新不会删除 Vault、分页偏好、AI 配置、自动填充设置或仍有效的指纹凭据。

## 发布阻断

以下任一情况阻断发布：

- 连续后台返回并认证后，正文高度仍被压缩或出现半屏白屏。
- 指纹弹窗与软键盘同时出现、循环弹出，或返回后界面无法交互。
- 首次创建 Vault 时主密码输入框不再自动聚焦。
- 条目编辑、搜索或其他正常输入页面的键盘布局被全局禁用或遮挡。
- 覆盖安装导致 Vault、偏好、自动填充设置或指纹凭据丢失。
