# SecretBase V3.3 macOS arm64 真机验收清单

本清单只使用空数据目录或专用测试数据，不读取生产 vault。全部阻断项通过后，才允许创建 `v3.3.0` 标签。

## 环境与产物

- [ ] Apple Silicon Mac，macOS 13 或更高版本。
- [ ] DMG、ZIP 和 `SHA256SUMS.txt` 均来自同一次 GitHub Actions 构建。
- [ ] DMG 与 ZIP 的 SHA-256 与校验文件一致。
- [ ] 应用显示为 arm64，Bundle ID 为 `io.github.langxi13.secretbase`。
- [ ] 发布包中不存在 vault、备份、日志、设置或本机绝对路径。

## 安装与首次启动

- [ ] DMG 可以挂载，并能将 SecretBase 拖入 Applications。
- [ ] ZIP 解压后得到完整 `SecretBase.app`。
- [ ] 未签名提示可以通过“系统设置 -> 隐私与安全性 -> 仍要打开”完成确认。
- [ ] 启动后没有额外终端窗口，界面由 WKWebView 正常显示。
- [ ] 应用数据写入 `~/Library/Application Support/SecretBase`，不写入应用包或 DMG。

## 桌面生命周期

- [ ] 重复启动只保留一个实例，并恢复聚焦已有窗口。
- [ ] 关闭窗口时只显示退出和取消，不出现托盘选项。
- [ ] 取消关闭后应用保持可交互，vault 不被锁定。
- [ ] 确认退出后窗口、FastAPI 和 Unix Socket 全部结束。
- [ ] `Command + Q` 可以正常退出且不遗留进程。
- [ ] 异常启动时显示 macOS 错误提示，不出现 WebView2 文案。

## 功能与界面

- [ ] 初始化、解锁、自动锁定和修改主密码正常。
- [ ] 条目、标签、密码组、分页、搜索、回收站和备份恢复正常。
- [ ] 原生文件选择框可以完成加密导出、明文导出和导入。
- [ ] 桌面状态正确显示 macOS、arm64、WKWebView 和平台能力。
- [ ] 数据、备份和日志目录入口正常，诊断摘要不包含真实用户名和主目录。
- [ ] 手动检查更新正常，断网失败不影响密码库功能。
- [ ] 自由调整窗口至窄屏后，正文和弹窗仍可滚动，按钮和文字不重叠。
- [ ] 短高度桌面窗口的左侧栏不显示重复管理区，并能看到“更多标签”和至少两个常用标签。
- [ ] `Command + +`、`Command + -`、`Command + 0` 可调整和重置比例，提示显示实际百分比。
- [ ] 重启应用后恢复最后缩放比例；普通加减号输入不触发缩放。
- [ ] macOS 设置页不显示“隐藏到托盘”，并显示删除应用后默认保留本地数据。

## 跨平台迁移

- [ ] Windows 导出的加密备份可在 macOS 导入，条目、标签、密码组和隐藏字段一致。
- [ ] macOS 导出的加密备份可在 Windows 导入，数据语义一致。
- [ ] 删除 `SecretBase.app` 后用户数据保持不变，重新安装后仍可解锁。
- [ ] 将 `.app` 移入废纸篓无需独立卸载器；手动删除应用支持目录前已有可恢复的加密备份。
