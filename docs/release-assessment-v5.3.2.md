# SecretBase V5.3.2 发布评估

## 结论

V5.3.2 是 V5.3 的交互可靠性正式维护版本。代码和自动化门禁已经完成，正式资产由 `v5.3.2` 标签触发的 Release 流水线统一构建、签名和校验。版本不改变 Vault V1 数据格式、V2 加密快照协议、AI 隐私黑名单、默认数据目录、发布签名或已有同步空间，可以作为 V5.3.1 的覆盖升级版本。

## 本次修复

- 锁定、托盘隐藏、401 会话失效和重新解锁时，前端会取消旧请求并清理所有敏感状态；旧请求不能在解锁后覆盖新会话。
- 条目、标签、密码组、备份和回收站加载失败时保留可用数据并提供重试；网络故障不会显示误导性的空数据状态。
- 确认弹窗支持失败后留在原处重试，并防止旧异步回调关闭新弹窗；备份密码和导入密码改为应用内输入弹窗。
- 分页数量统一做合法化处理，切换数量后回到第一页；回收站删除最后一页项目后自动校正页码，备份和回收站提供处理中反馈。
- 密码组排序、星标、批量操作和详情/编辑请求增加并发保护，快速连续点击不会产生重复写入或错位显示。
- AI 专业工具请求期间冻结会改变上下文的控件；AI 管家会忽略关闭后的过期历史响应，清除 AI 配置需要二次确认。
- Store 读取在控制器和 Store 两层检查请求代际；锁定或会话切换后的迟到条目、标签和密码组响应不会写入内部状态。
- 桌面更新轮询采用单飞策略，慢响应不会形成并发轮询；Android AI 设置清除始终释放活动锁，AI 管家和专业工具可取消网络请求并忽略迟到结果。
- 同步历史读取失败提供原地重试，锁定时停止 AI 诊断轮询并清除同步主机、恢复码和应用密码等状态。
- 移动同步的加入、同步、冲突解决、历史恢复、历史压缩和密钥轮换把远端写入与本地状态刷新分开处理；远端成功而刷新失败时保留成功提示并给出重新读取入口。
- 移动 AI 增加取消待发送请求和放弃建议入口；建议审核期间锁定工具、设置和输入上下文，删除当前对话需要确认并回到新会话。
- 回收站、标签管理、备份恢复和桌面更新补充关闭、翻页、切换和重复触发锁，降低迟到响应覆盖当前界面的风险。
- 修复设置页获取 AI 模型后关闭页面可能永久停留在“获取中”的状态机问题，并增加过期响应回归保护。
- 修复多层弹窗的 Escape 处理顺序，子弹窗关闭不会误关底层设置或留下错误的界面层级。
- 锁定和关闭同步界面时清除连接草稿、主机信息、恢复码及历史上下文；Android 主导航返回会先恢复条目筛选上下文。
- 导入预览/冲突界面直接展示失败信息，用户可在原地重试或取消。
- 同步加入表单把连接测试与恢复码校验分开；缺少恢复码时按钮仍可点击并在弹窗内说明原因，同时通过顶层 Toast 提示，不再出现“测试成功但按钮灰掉”的无解释状态。
- Web 静态资源使用独立修订号 `5.3.2-web-20260719-3`，适配 Nginx 的长缓存策略，确保同步 UI 修复不会被旧 JavaScript 或模板覆盖。
- 弹窗正文与底部操作栏采用统一的 flex 滚动契约；直接承载表单的弹窗会让表单参与剩余高度分配，避免长恢复码、配对链接和确认字段被按钮遮挡。AI 范围、建议关联条目和发送确认层的长列表也具有独立滚动及移动端安全区。

## 安全边界

- 已有字段值不会因为本次交互修复而进入普通 AI 请求；AI 黑名单仍由服务端拒绝，客户端收到黑名单响应也不会调用删除接口。
- 请求取消只发生在本机会话边界，不会修改远端同步协议或跳过 WebDAV TLS 校验。
- 本次没有读取、写入或提交 `backend/data`、备份目录、环境变量、主密码、WebDAV 密码或个人订阅信息。
