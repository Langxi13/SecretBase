# SecretBase v3.0.0 发布评估

## 结论

SecretBase 已具备单用户本地加密密码库的完整核心闭环，可以发布 `v3.0.0` 作为 Web 稳定版和桌面基础模式基线。

本结论不等于项目不存在后续工作。当前版本适合个人自托管和源码本地运行；Windows 原生桌面窗口、安装包、签名和自动更新仍属于后续版本。

## 已验证范围

- 主密码初始化、解锁、锁定、修改密码和自动锁定。
- 条目 CRUD、隐藏/复制字段、星标、搜索、排序、筛选和分页。
- 标签与密码组创建、编辑、排序、批量归属和 AI 建议确认。
- 回收站、批量操作、导入导出、手动/自动备份和恢复向导。
- AI 配置加密保存、解析、整理和结构化操作计划。
- vault 文件锁、乐观冲突、原子写入和写入前备份。
- 服务端 nginx 部署与桌面同源静态托管。
- Windows/Linux 本地启动引导和桌面数据路径隔离。
- 最新允许依赖组合与项目固定依赖组合。

完整自动化检查由 `python scripts/run-release-checks.py` 执行，并在 GitHub Actions 的 Linux 与 Windows 环境持续验证。

## 本次收口的风险

- 前端执行代码不再依赖 Vue CDN，桌面运行时可以离线启动。
- 移除 Google Fonts 和第三方 favicon 请求，避免泄露条目域名与使用信息。
- 认证表单增加 POST 无脚本降级，并在页面最早阶段清理遗留敏感查询参数。
- 桌面模式限制 CORS 与 Host，仅允许当前本机来源。
- 后端统一返回防嵌入、禁止 referrer 和权限限制安全头。
- 外部网址使用 `noopener,noreferrer` 打开。
- Python 直接依赖固定版本，降低首次本地安装随上游漂移而失效的风险。
- 增加跨平台一键启动、CI 和标签发布工作流。

## 仍需关注的边界

### 产品边界

- 当前仍是单用户、单 vault，不提供团队协作和账号找回。
- 没有浏览器扩展、自动填充、云同步和跨设备实时同步。
- 密码生成器尚未作为独立工作流交付。
- AI 能力依赖用户选择的外部服务商，发送范围仍需用户自行评估。

### 桌面边界

- V3.0 使用浏览器作为窗口，不是独立桌面应用。
- 浏览器关闭不会自动停止本地后端，需要在控制台结束进程。
- 尚未实现多实例协调、托盘、安装器、签名和自动更新。
- Windows `.cmd`/PowerShell 启动已进入 CI，但最终 exe 必须在 Windows 真机手动验收。

### 规模与平台边界

- 当前分页和测试覆盖适合个人 vault；超大规模条目仍需专门压测。
- 移动端已有响应式 Web 布局，但不等同于 iOS/Android 原生应用。
- macOS 当前只承诺源码本地运行，正式打包应在 Windows MVP 稳定后推进。

## 后续路线

1. V3.1：Windows one-folder 便携版，PyInstaller + pywebview。
2. V3.2：Windows 安装器、错误页、单实例、托盘和可选签名。
3. V3.3：macOS 桌面打包与签名/公证评估。
4. V4.0：固化 vault 规范与跨语言兼容测试，为移动端共享核心做准备。
5. V5.0：iOS/Android 离线应用，首版通过加密备份迁移数据。

## 数据安全确认

- 本次没有修改 vault 文件格式。
- 没有对真实条目、标签、密码组或备份执行自动迁移。
- 自动测试只使用临时测试 vault。
- 生产部署只需要代码更新、服务重启和只读健康检查。
