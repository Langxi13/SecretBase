# SecretBase V3.2 Windows 桌面产品化

本文档记录 V3.2 的实现边界和发布门禁。V3.2 在 V3.1 Windows one-folder 便携版基础上增加安装、诊断和托盘体验，不改变 vault 格式、Web 部署方式或主密码安全模型。

状态：功能分支实现阶段。代码、Linux 回归和静态检查完成后，仍需 Windows Server 2022/2025 CI 与 Windows 10/11 真机验收，才能创建正式 `v3.2.0` 标签。

## 1. 交付目标

V3.2 同时提供：

```text
SecretBase-v3.2.0-windows-x64-setup.exe
SecretBase-v3.2.0-windows-x64.zip
SHA256SUMS.txt
```

- GitHub Release 将安装器作为独立 `.exe` 资产发布；日常使用直接下载安装器，不需要先下载或解压 ZIP。
- 便携 ZIP 仅作为免安装和故障排查备用，保持 V3.1 使用方式。GitHub Actions 测试产物受平台限制仍会带一层外部 ZIP，不作为正式用户下载入口。
- 两种形态都读取 `%LOCALAPPDATA%\SecretBase`，已有 vault 不需要迁移。
- 桌面模式新增状态诊断、目录入口、手动检查更新和可选系统托盘。
- 服务端模式不显示桌面入口，原有 API、前端和部署方式保持兼容。

## 2. 安装和卸载

- 使用 Inno Setup 6.7.1，固定 AppId，默认安装到 `%LOCALAPPDATA%\Programs\SecretBase`。
- 安装器只面向当前 Windows 用户，不请求管理员权限，不增加开机启动。
- 覆盖安装和升级只替换程序文件，不修改 `%LOCALAPPDATA%\SecretBase`。
- 默认卸载只删除程序、快捷方式和安装登记，保留 vault、备份、设置和日志。
- 用户勾选删除本地数据并输入 `DELETE` 后，才删除整个 `%LOCALAPPDATA%\SecretBase`。
- 静默卸载只有同时提供 `/PURGEDATA=1 /CONFIRMDELETE=DELETE` 才清除数据。

## 3. 桌面状态与更新

初始化页、锁定页和“设置 -> 桌面”可以打开桌面状态：

- 显示当前版本、安装版/便携版、Windows 版本和 Edge WebView2 渲染器。
- 检查本地后端、数据目录、备份目录和日志目录是否正常可写。
- 只允许打开固定的数据、备份和日志目录，不接受任意路径。
- 可复制脱敏诊断摘要；摘要不包含 Windows 用户名、真实绝对路径、条目、token 或 API Key。

“检查更新”只在用户点击后访问 SecretBase 的 GitHub 最新正式 Release：

- 不在启动时联网。
- 不发送 vault 内容或本机数据。
- 只比较版本并打开系统浏览器下载页，不自动下载或替换程序。
- 网络不可用不影响任何密码库功能。

## 4. 可选系统托盘

`close_to_tray` 默认 `false`，`confirm_close` 默认 `true`。桌面窗口最小尺寸为 `360 × 320`，窄窗口使用既有移动端布局。关闭流程要求：

- 默认点击关闭按钮时先显示应用内确认框，提供隐藏到托盘、完全退出和取消三个明确操作。
- pywebview 原生关闭回调只取消本次关闭并调度后续动作，不得在回调内同步调用 JavaScript、启动托盘或锁定 vault。
- 前端关闭按钮调用的 Python bridge 必须先返回，再异步隐藏窗口或销毁窗口，禁止 Python -> JavaScript -> Python 的同步重入。
- 勾选“不再提醒”后将本次操作写入 `close_to_tray`，同时把 `confirm_close` 保存为 `false`；设置页必须允许恢复提醒。
- 保存关闭设置只更新偏好，不得在 pywebview 桥接调用中立即创建托盘线程；托盘仅在真正隐藏窗口时按需启动。
- 选择隐藏到托盘后先锁定 vault、清除会话 token，再隐藏窗口。
- 前端必须在窗口隐藏前立即清空条目、标签、密码组、已显示字段和弹窗状态，恢复时直接显示解锁页。
- 最小化按钮继续保留在任务栏，不进入托盘。
- 托盘菜单固定提供“打开 SecretBase”“锁定”“退出”。
- 再次启动 SecretBase 会激活已有窗口。
- 显式退出会停止托盘、本地后端和桌面进程。
- 托盘初始化失败时保持窗口可见并显示错误，禁止留下不可见后台进程。
- 安装、升级和卸载通过独立退出信号关闭已有实例，不把卸载器的关闭请求重新隐藏到托盘；仅在旧测试版不支持退出信号时使用系统进程终止作为兼容兜底。

## 5. 构建和签名

Windows 构建环境需要 Python 3.11 x64 和 Inno Setup 6.7.1：

```powershell
.\scripts\build-desktop-windows.ps1
```

构建脚本会完成 PyInstaller 打包、运行时自检、可选签名、ZIP、安装器和两项 SHA-256。桌面依赖固定 `pystray`、Pillow、six、pythonnet 和 clr-loader 版本。

可选签名环境变量：

```text
WINDOWS_SIGNING_CERT_BASE64
WINDOWS_SIGNING_CERT_PASSWORD
WINDOWS_SIGNING_TIMESTAMP_URL
```

证书和密码都未配置时生成未签名产物；只配置其中一项时构建失败。签名证书不会写入仓库或发布包。未签名安装器可能触发 Windows SmartScreen 提示。

## 6. 自动化和真机门禁

Windows CI 必须：

1. Windows Server 2022 构建便携 ZIP 和安装器。
2. 验证两个产物的 SHA-256 和敏感文件边界。
3. Windows Server 2025 下载同一产物并复测 CLR、WinForms、WebView2、托盘依赖和便携包 Mark-of-the-Web。
4. 确认安装器可携带浏览器下载标记；解除标记后执行无人值守安装和打包自检。未签名安装器的 SmartScreen 人工确认由真机清单覆盖。
5. 默认卸载后确认用户数据仍存在。
6. 重新安装并使用双确认参数卸载，确认数据目录被删除。
7. 保持一个 SecretBase 实例运行时执行卸载，确认卸载器能先结束该实例且不要求任务管理器介入。
8. 确认所有 SecretBase 进程退出。

正式发布前还必须完成 [V3.2 Windows 真机验收清单](manual-qa-checklist-v3.2.md)。

## 7. 非目标

V3.2 不包含应用内自动更新、保存主密码、开机启动、全机管理员安装、浏览器自动填充、云同步、macOS 包或 vault 格式迁移。
