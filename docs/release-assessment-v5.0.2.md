# SecretBase V5.0.2 发布评估

## 结论

V5.0.2 是不改变 Vault V1、数据目录或签名密钥的跨端维护版本。发布重点是修复移动端 AI 复合计划差异、生产 AI 代理超时、前端受限存储与错误响应、缓存版本维护和默认 Rust 测试命令。

## 已审计范围

- 后端认证、条目、标签、密码组、回收站、备份、导入导出、设置、工具报告和 AI 路由。
- Web/Vue 初始化、锁定、分页、筛选、分类刷新、弹窗写入锁、AI 确认与撤回、桌面桥和更新状态。
- Windows/macOS 桌面生命周期、托盘、关闭确认、缩放、单实例、打包隐私扫描和更新安装交接。
- Android Vault、生命周期保护、生物识别、分页、导航返回、AI 隐私、复合计划、撤回和签名更新。
- Vault V1 Python/Rust 黄金向量、未知字段保留、认证失败统一和发布资产边界。

## V5.0.2 关键结果

- Android AI 管家允许跨领域复合计划，同时由本机执行排序和冲突检测。
- 浏览器存储不可用时应用安全降级；非 JSON HTTP 错误不再被误分类。
- 默认搜索范围可直接使用且不包含字段值。
- 全部前端资源使用统一应用版本缓存参数，入口和运行时配置禁止缓存。
- 全部 AI 反向代理接口使用 180 秒超时。
- 业务日志移除条目标题、标签名和密码组名。
- 首次主密码、批量操作、分类排序和旧 AI 计划接口补齐显式输入上限，同时保留已有超长主密码的解锁兼容性。
- 标准 `cargo test --locked` 与全 feature Vault 向量都进入 CI。
- GitHub Dependabot 漏洞告警已启用；发布前已升级 Python 与 Rust 告警依赖，目标为开放告警归零。

## 发布门禁

- Python 编译、后端单元测试和完整 API 假数据冒烟。
- 全部前端 Node 运行时契约与 JavaScript 语法检查。
- Flutter analyze 与串行 Widget/状态测试。
- Mobile Rust、Vault Core 默认/全 feature 测试、fmt 和 Clippy。
- Windows、macOS、Android CI 构建、下载后产物自检和签名更新清单验证。

## 仍需真机确认

- Windows 5.0.1 到 5.0.2 原地覆盖后的托盘、缩放和数据保留。
- Android 5.0.1 到 `versionCode=5000200` 的正式签名覆盖、生物识别和复合 AI 审核。
- macOS DMG 手动覆盖后的应用支持目录保留。

下一规划版本为 V5.1.0，建议只开展 iOS 工程、签名、分发和安全存储可行性，不在 V5.0.2 中引入新的 Vault 格式或云同步能力。
