# SecretBase V5.3.0 人工验收清单

## 1. 基础与升级

1. 在 Windows、macOS、Android 分别覆盖安装，确认原 Vault、备份、AI 配置、自动填充和生物识别行为不被破坏。
2. 锁定后点击同步入口，确认先完成主密码/指纹解锁，不会卡死或显示旧会话。
3. 断网、切后台、恢复网络后重试，确认按钮状态可恢复，界面不出现空白或重复提交。

## 2. 创建与加入

1. 在专用测试 WebDAV 目录创建 V2 空间，检查能力测试包含 `PROPFIND` 和基础读写，不要求 ETag。
2. 显示恢复码前输入错误主密码，确认只提示主密码错误，不泄露同步密钥。
3. 在第二台设备用恢复码加入，确认空 Vault 可加入；已有数据时必须显式勾选合并。
4. 使用错误地址、错误应用密码、HTTP 公网地址和证书错误，确认提示可操作且不会反复重试。
5. 在已有本机内容的设备加入并人为中断首次合并快照上传，确认页面仍显示已加入；恢复网络后手动同步可继续上传，不会残留加入表单或 pending token。

## 3. 正常同步

1. A 端新建、修改、删除条目，B 端分别执行同步，确认数据和回收站状态一致。
2. 修改标签、密码组简介、排序和条目成员关系，确认元数据同步。
3. 重复点击同步、上传中关闭应用、上传后立即再次同步，确认不会覆盖快照或生成损坏对象。
4. 在 WebDAV 后台确认只存在加密 `.sbs` 对象，搜索密文不得出现测试字段值。
5. 修改 WebDAV 地址、账号、应用密码和设备名，确认连接字段变化会验证远端；仅切换自动同步或设备名时不强制联网。
6. 启用自动同步后修改 Vault，确认约 5 秒后只触发一次；自动同步进行中点击手动同步不会并发进入。锁定 Vault 时未完成请求应被取消。

## 4. 并发与冲突

1. 两台设备从同一基线修改同一条目，确认只显示标题、时间、字段数量和变化区块。
2. 分别选择“本机”“远端”“保留两份”，确认结果符合选择，保留两份会生成冲突副本。
3. 三台设备从同一基线分别修改同一条目，确认第一次处理后仍显示下一批冲突，第三个分支不会丢失。
4. 在冲突窗口期间修改本机 Vault 或让远端产生新快照，确认旧 token 失效并要求重新同步。

## 5. 历史、轮换与清理

1. 打开历史并恢复旧快照，确认恢复以新快照发布，不改写旧对象。
2. 轮换同步密钥后，旧恢复码和旧设备加入失败，新恢复码可用。
3. 未同步本机修改时尝试压缩，确认操作被拒绝。
4. 确认所有设备同步后输入 `COMPACT`，检查生成新根快照、旧空间逐对象清理、界面显示新恢复码。
5. 模拟清理阶段断网，确认新空间仍可同步，状态明确提示旧空间待清理。
6. 在两端内容完全相同但顶层条目顺序不同的测试 Vault 上执行同步和压缩，确认按条目 ID 集合判断为同一快照，同时保留每个条目的字段顺序。
7. 在旧空间中放入非协议对象或清理期间并发写入新对象，确认清理停止且不误报成功；本机 Vault 和新空间保持可用。
8. 说明并验证弱 WebDAV 清理不是目录树原子事务：即使重新枚举，空目录检查与删除之间仍可能竞态，界面只报告实际结果。

## 6. Android 专项

1. Android 10、较新 Android 设备分别创建/加入同步，确认指纹只解锁本机，不进入快照。
2. 切后台、锁屏、系统回收后重新进入，确认需要重新认证且布局完整，不出现下半屏白屏。
3. 在无 ETag、带 `DAV:` 前缀 XML、临时 503/429 的服务上测试，确认有限重试和错误提示。
4. 压缩历史后从旧设备打开，确认提示重新加入而不是覆盖旧空间。
5. 分别执行恢复码显示、历史恢复、轮换、压缩和远端删除，确认主密码错误时被拒绝；轮换/压缩/删除的确认词或一次性令牌不能重复使用。
6. 同步进行中锁定应用，确认活动请求取消、敏感表单清空，重新解锁后可正常重试且不出现半屏白屏。

## 7. 发布与隐私

1. 检查发布包和 Git 历史，不包含真实 WebDAV 地址、用户名、应用密码、服务器路径、Vault、日志或备份。
2. 校验 Windows 安装器、macOS DMG/ZIP、四种 Android APK、更新清单签名和 `SHA256SUMS.txt`。
3. 刷新 Web 前端并检查资源版本为 V5.3.0，无 `ReferenceError`、500、未定义响应式变量或旧缓存 UI。
