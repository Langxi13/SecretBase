# V5 跨端更新人工验收清单

## V5.1.0 功能版本回归

- [ ] Windows `5.0.2` 可以发现 `5.1.0`，覆盖安装后 Vault、偏好、托盘、缩放和 WebView 数据保留。
- [ ] Android `5.0.2` 可以发现 `versionCode=5010000`，永久签名覆盖安装后 Vault、指纹凭据、分页偏好和 AI 配置保留。
- [ ] macOS `5.0.2` 显示 `5.1.0` DMG，手动覆盖 `.app` 后应用支持目录保留。
- [ ] Android 覆盖升级后可在系统设置中启用 SecretBase 自动填充，并完成 Chrome、原生登录页和新密码页回归。
- [ ] Web/桌面从已有条目的密码组详情页新建条目时，当前密码组仍默认选中。

## V5.0.2 维护回归

- [ ] Windows `5.0.1` 可以发现 `5.0.2`，覆盖安装后 Vault、偏好、托盘和缩放状态保留。
- [ ] Android `5.0.1` 可以发现 `versionCode=5000200`，系统覆盖安装后 Vault、生物识别和分页偏好保留。
- [ ] macOS `5.0.1` 显示 `5.0.2` DMG，手动覆盖 `.app` 后应用支持目录保留。
- [ ] Android AI 管家请求“新建一个标签和一个密码组”时显示按领域分组的同一审核计划，一次应用并提供撤回。
- [ ] Android AI 计划同时删除并分配同一标签时，本机拒绝冲突选择且不写入 Vault。
- [ ] Web 默认搜索无需先选范围即可命中标题；字段值只有手动勾选后才参与搜索。
- [ ] 禁用浏览器站点存储后仍可初始化、解锁和操作；刷新后允许重新登录，不出现白屏。
- [ ] Nginx 部署下 AI 对话超过 60 秒但未超过上游 120 秒时不被代理提前中断。

## Windows 安装版

- [ ] 从旧安装版覆盖安装 V5 基线，确认 `%LOCALAPPDATA%\SecretBase` 中的 Vault、设置和备份仍存在。
- [ ] 自动检查不阻塞启动、解锁和离线操作；关闭自动检查后重启仍保持关闭。
- [ ] 尚无包含签名清单的正式 Release 时显示中性提示，不暴露 HTTP 404；系统时间或代理证书异常时提供可操作的 TLS 提示。
- [ ] 下载过程显示进度且可取消，断网或哈希错误不会出现“立即更新”。
- [ ] 确认安装后先锁定 Vault，再退出托盘和窗口；安装完成只启动一个新实例。
- [ ] 更新失败后旧版本仍能启动，更新日志和重试入口可用。
- [ ] 便携版只显示官方链接，不尝试修改自身目录。

## Android

- [ ] 临时签名 CI 包明确提示先备份并迁移，不尝试覆盖安装正式包。
- [ ] 永久签名基线可以通过系统安装界面覆盖到更高 `versionCode`，私有 Vault 和偏好保留。
- [ ] Wi-Fi 默认自动预下载；移动网络默认只提示，开启流量选项后才自动下载。
- [ ] 尚无包含签名清单的正式 Release 时显示中性提示，不把 HTTP 404 显示为更新故障。
- [ ] 未授予“安装未知应用”时进入 SecretBase 专属授权页，返回后可继续安装。
- [ ] 错误包名、低版本、错误签名、错误大小和错误 SHA-256 均被拒绝。
- [ ] 安装前 Vault 锁定，取消系统安装后重新进入应用仍要求解锁。

## macOS

- [ ] 自动检查能显示统一版本和正式 DMG 地址。
- [ ] 安装包携带 CA 证书集，标准 GitHub HTTPS 证书可通过校验；尚无正式清单时显示中性提示。
- [ ] 应用不会尝试用未签名助手替换 `/Applications/SecretBase.app`。
- [ ] 手动拖入并覆盖新 `.app` 后，应用支持目录中的 Vault、设置和日志保持不变。

## 发布门禁

- [x] 普通 CI APK 文件名带 `-ci`，正式 Release 拒绝临时签名产物。
- [x] `release` 环境审批前，Android 构建和清单签名任务不能读取永久密钥。
- [x] Release 同时包含三端产物、`SHA256SUMS.txt`、签名清单及签名文件。
- [x] 修改清单任意字节后，桌面和 Android 客户端都拒绝更新。
